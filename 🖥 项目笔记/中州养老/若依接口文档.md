# 利用若依生成在线接口文档

## 方法一：利用Swagger进行接口调试

### 一、**若依项目中与Swagger配置有关的文件：**

 1. 后端配置文件：application.yml

 2. 后端核心配置类：zzyl-admin/src/main/java/com/zzyl/web/core/config/SwaggerConfig

 3. 前端映射地址：src/views/tool/swagger/index.vue

    ```javascript
    const url = ref(import.meta.env.VITE_APP_BASE_API + "/swagger-ui/index.html")
    ```

    

### **二、导入依赖**

Spring已经将Swagger纳入自身的标准，建立了Spring-swagger项目，现在叫Springfox。通过在项目中引入Springfox ，就可以非常简单快捷的使用Swagger。

```xml
<!-- swagger3-->
<dependency>
	<groupId>io.springfox</groupId>
	<artifactId>springfox-boot-starter</artifactId>
</dependency>

<!--防止进入swagger页面报类型转换错误，排除3.0.0中的引用，手动增加1.6.2版本--><dependency>
	<groupId>io.swagger</groupId>
	<artifactId>swagger-models</artifactId>
	<version>1.6.2</version>
</dependency>
```



### 三、访问地址

方式一：系统内部访问：“系统工具”——“系统接口”

方式二：服务地址 + /swagger-ui/index.html ——> http://localhost:8080/swagger-ui/index.html



### 四、注意点

1. Swagger无论是站内访问还是站外访问，都是直接访问服务器，不需要加地址前缀 dev-api
2. 401没有权限：拷贝token至 swagger Authorization中



## 方法二：利用Knife4j生成接口文档

### 一、**若依项目中与Knife4j配置有关的文件：（与Swagger相同）**

 1. 后端配置文件：application.yml

 2. 后端核心配置类：zzyl-admin/src/main/java/com/zzyl/web/core/config/SwaggerConfig

 3. 前端映射地址：src/views/tool/swagger/index.vue

    ```javascript
    // 注意映射地址要改成 doc.html
    const url = ref(import.meta.env.VITE_APP_BASE_API + "/doc.html")
    ```

    

### 二、导入依赖

```xml
<!-- swagger knife4j配置 -->
<dependency>
	<groupId>com.github.xiaoymin</groupId>
		<artifactId>knife4j-spring-boot-starter</artifactId>
	<version>3.0.3</version>
</dependency>
```



### 三、访问地址

方式一：系统内部访问：”“系统工具”——“系统接口”（前提是vue的映射地址改为了“doc.html”）

方式二：http://localhost:8080/doc.html



### 四、注意点

1. Knife4j 的站内访问请求的是前端服务器，需要加地址前缀dev-api；站外访问请求的是后端服务器，不需要加地址前缀 dev-api。设置是否加前缀的代码在后端核心配置类里。
2. 401没有权限：拷贝token至 swagger Authorization中



### 五、提高接口文档可读性（给接口文档中的类、方法、参数打上中文说明）

#### 5.1 常见注解（打在Controller层）

| 注解              | 说明                                                         |
| ----------------- | ------------------------------------------------------------ |
| @Api(tags = "")   | 用在类上，描述Controller的作用                               |
| @ApiOperation     | 用在方法上，说明方法的用途、作用                             |
| @ApiParam         | 用在方法的参数上，描述单个形参的含义，适用于简单场景         |
| @ApiImplicitParam | 用在方法上，描述单个形参的含义，适用于参数复杂或者详细描述参数的场景 |
| @ApiModel         | 用在类上，用对象来接收参数或者返回参数，描述类的含义         |
| @ApiModelProperty | 用在类的属性上，用对象来接收参数或者返回参数，描述字段的含义 |

<span style="color:red">注意@Api注解里面要加上 `tags =`</span>



#### 5.2 复杂注解示例

#### **5.2.1 @ApiImplicitParams + @ApiImplicitParam 给实体类属性添加说明**

```java
@ApiOperation("新增用户")
@ApiImplicitParams({
	@ApiImplicitParam(name = "userId", value = "用户id", dataType = "Integer", dataTypeClass = Integer.class),
    @ApiImplicitParam(name ="username", value = "用户名称", dataType = "String", dataTypeClass = String.class),
    @ApiImplicitParam(name = "password", value = "用户密码", dataType = "String", dataTypeClass = String.class),
    @ApiImplicitParam(name = "mobile", value = "用户手机", dataType = "String", dataTypeClass = String.class)
})
public R<String> save(@ApiParam("用户信息实体")@RequestBody UserEntity user){方法体}
```



#### **5.2.2 @ApiModel + @ApiModelProperty 给实体类属性添加说明**

```java
@ApiModel(value = "UserEntity", description = "用户实体")
public class UserEntity{
	@ApiModelProperty ("用户ID")
    sprivate Integer userId;
	@ApiModelProperty("用户名称")
    private String username;
	@ApiModelProperty("用户密码")
    private String password;
	@ApiModelProperty("用户手机")
    private String mobile;
}
```



#### **5.2.3 给有泛型的对象添加说明**

方法：给有泛型的对象规定类型，把 <?> 全部替换成 <T>，然后在方法返回值类型里显性指明类型

```java
@ApiModel("分页结果集实体")
public class TableDataInfo<T> implements Serializable{}

//  =============

public TableDataInfo<NursingProject> list(参数列表){方法体}
```



#### 5.2.4 给AjaxResult添加说明

由于AjaxResult 的类型是一个Map而不是类，所以无法直接给里面的键添加说明，但是若依里面还有一个AjaxResult 的相似替代类：R，R的属性与AjaxResult 的键都一样，但是R是一个类，它可以指定泛型。

记住两句公式：

​	返回成功：AjaxResult.success() = R.ok();

​	返回失败：AjaxResult.error() =R.fail();

```java
// 原生 AjaxResult 代码
return toAjax(nursingProjectService.insertNursingProject(nursingProject))

// 改写成等价的 R 代码
int rows = nursingProjectService.insertNursingProject(nursingProject);
return rows > 0 ? R.ok() : R.fail();
```
