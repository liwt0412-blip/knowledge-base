# 🔧 代码问题记录

记录项目代码扫描发现的潜在问题，方便后续修复。

---

## 2026-05-02（新增 Activities/Interceptor 模块扫描）

### 1. 🐛 ActivitiesMapper 声明为 class 而非 interface（编译错误）
- **文件**: `qk-management/src/main/java/com/qkgj/mapper/ActivitiesMapper.java` 第 12 行
- **问题**: `public class ActivitiesMapper {` — MyBatis Mapper 必须是 `interface`，声明为 class 后第 15 行的无实现方法 `listActivities()` 会导致编译报错
- **错误信息**: "Missing method body, or declare abstract"
- **应改为**: `public interface ActivitiesMapper {`

### 2. 🐛 ActivitiesMapper.xml resultType 写错
- **文件**: `qk-management/src/main/resources/mapper/ActivitiesMapper.xml` 第 4 行
- **问题**: `resultType="com.qkgj.entity.Course"` — 实体引用错误，应为 Activity
- **应改为**: `resultType="com.qkgj.entity.Activity"`

### 3. 🐛 ActivitiesMapper.xml SQL 语句为空
- **文件**: `qk-management/src/main/resources/mapper/ActivitiesMapper.xml` 第 4-6 行
- **问题**: `<select>` 标签之间没有任何查询 SQL，执行时会报 SQL 异常
- **建议**: 补充完整的查询语句，如 `select * from activity where ...`

### 4. 🐛 ActivityController @RequestParam 绑定复杂对象错误
- **文件**: `qk-management/src/main/java/com/qkgj/controller/ActivityController.java` 第 25 行
- **问题**: `@RequestParam ActivitiesDto activitiesDto` — `@RequestParam` 不能绑定 DTO 复杂对象，Spring 会把 `activitiesDto` 当成单个参数名去找，而不是把 `channel`/`status` 等字段分别绑定
- **应改为**: 去掉 `@RequestParam`，直接写 `ActivitiesDto activitiesDto`

### 5. ⚠️ TokenFilter 多余导入
- **文件**: `qk-management/src/main/java/com/qkgj/filter/TokenFilter.java` 第 3-4 行
- **问题**: 导入了 `cn.hutool.http.server.HttpServerRequest` 和 `HttpServerResponse` 但从未使用
- **建议**: 删除无用的 import

### 6. ⚠️ TokenFilter + TokenInterceptor 双重权限校验
- **文件**: `TokenFilter.java`（Filter） + `TokenInterceptor.java`（Interceptor）同时存在
- **问题**: 两个组件做同一件事（token 校验），请求经过 Filter 后又被 Interceptor 再次校验一次
- **建议**: 只保留一个，推荐保留 Interceptor（更灵活，WebConfig 中方便配置排除路径）

### 7. ⚠️ WebConfig 排除路径不足
- **文件**: `qk-management/src/main/java/com/qkgj/config/WebConfig.java` 第 16 行
- **问题**: Interceptor 只排除了 `/login`，但 SessionController 的演示接口 `/c1`、`/c2`、`/s1`、`/s2` 未排除
- **影响**: 访问这些 demo 接口会 401 报错
- **建议**: 排除路径改为 `.excludePathPatterns("/login","/c1","/c2","/s1","/s2")`

## 2026-04-29（项目初次扫描）

### 1. 🐛 getUserInDept SQL 查询写错
- **文件**: `qk-management/src/main/resources/mapper/UserMapper.xml` 第 94 行
- **问题**: 查询的是 `role` 表，应该查 `user` 表
- **原文**: `select * from role where id = #{id}`
- **应改为**: `select u.* from user u where u.dept_id = #{deptId}`
- **影响**: 按部门查询用户接口返回数据错误

---

### 2. ⚠️ 根 pom.xml 语法错误
- **文件**: `pom.xml` 第 2 行
- **问题**: `<project> xmlns=` 中间多了空格，maven 解析可能报错
- **原文**: `<project> xmlns="http://maven.apache.org/POM/4.0.0"`
- **应改为**: `<project xmlns="http://maven.apache.org/POM/4.0.0"`

---

### 3. ⚠️ PageHelper 版本冲突
- **文件**: `qk-management/pom.xml` 第 69 行
- **问题**: 父 POM 定义版本为 `2.1.0`，子模块又硬编码为 `1.4.6`
- **建议**: 子模块去掉 `<version>`，统一走父 POM 管理

---

### 4. 🐛 addCourse 方法用了 private
- **文件**: `qk-management/.../controller/CourseController.java` 第 48 行
- **问题**: `private Result addCourse(...)` 应该是 `public`
- **影响**: 添加课程接口外部无法访问，返回 404

---

### 5. ⚠️ RoleDto 字段名拼写错误
- **文件**: `qk-entity/.../dot/RoleDto.java` 第 9 行
- **问题**: `lable` 应为 `label`
- **影响**: 如果 Role 实体也用这个字段查询，会映射不上

---

### 6. ⚠️ Activity 实体类无业务代码
- **文件**: `qk-entity/.../entity/Activity.java`
- **问题**: 实体已建（有 channel/startTime/endTime/discount/voucher 等字段），但没有任何 Controller/Service/Mapper
- **建议**: 要么删除，要么补充完整增删改查

---

### 7. ⚠️ 全局异常处理器不完整
- **文件**: `qk-management/.../exception/GlobleExceptionHandler.java`
- **问题**: 通用异常 `Exception` 的兜底处理被注释掉了，只处理 BusinessException
- **影响**: 非业务异常（如空指针、SQL 异常）不会返回友好提示，直接 500 报错

---

### 8. ⚠️ LoginVo 注解搭配问题
- **文件**: `qk-entity/.../vo/LoginVo.java`
- **问题**: `@Data` + `@Builder` 同时使用需要再加 `@AllArgsConstructor`
- **影响**: 编译不报错但运行时有潜在问题（缺少全参构造器）

---

### 9. ℹ️ 其他小问题
- `Demo.java` 中有硬编码的 Windows 本地路径 (`C:\Users\ALIENWARE\Desktop\...`)
- 日志配置中 FILE 文件输出被注释掉，只有控制台日志
- 部分 Mapper 用注解，部分用 XML，风格不统一
