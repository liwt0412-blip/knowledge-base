# QK-CRM 接口文档

> 项目：qk-parent  
> 基础路径：`http://localhost:9090`  
> 统一响应格式：Result 风格

---

## 一、统一响应格式

### 1.1 成功响应

```json
{
  "code": 1,
  "msg": "success",
  "data": { ... }
}
```

### 1.2 失败响应

```json
{
  "code": 0,
  "msg": "错误信息描述",
  "data": null
}
```

### 1.3 分页响应

data 字段内为 PageResult 结构：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "total": 100,
    "rows": [ ... ]
  }
}
```

### 1.4 公共请求头

| 头字段 | 说明 | 必填 |
|--------|------|------|
| Authorization | Bearer token（登录接口返回的 token） | 是（登录接口除外） |

## 三、用户管理

### 3.1 分页查询用户

`GET /users`

**请求参数（Query）：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| name | String | - | 姓名（模糊） |
| phone | String | - | 手机号 |
| status | Integer | - | 状态 1:正常 0:停用 |
| deptId | Integer | - | 部门ID |
| page | Integer | 1 | 页码 |
| pageSize | Integer | 5 | 每页条数 |

**响应 data：** PageResult\<User\>

### 3.2 新增用户

`POST /users`

**请求体：**

```json
{
  "username": "zhangsan",
  "password": "123456",
  "name": "张三",
  "phone": "13800138000",
  "email": "zs@qq.com",
  "gender": 1,
  "status": 1,
  "deptId": 1,
  "roleId": 2,
  "remark": "备注"
}
```

### 3.3 根据ID查询用户

`GET /users/{id}`

**响应 data：** User 对象

### 3.4 修改用户

`PUT /users`

**请求体：** User 对象（带 id）

### 3.5 删除用户

`DELETE /users/{ids}`

**说明：** ids 支持批量删除，逗号分隔

### 3.6 查询所有用户

`GET /users/list`

**说明：** 返回所有用户列表，不分页

### 3.7 根据角色查询用户

`GET /users/role/{roleLabel}`

**示例：** `/users/role/sales` → 返回所有销售角色用户

### 3.8 根据部门查询用户

`GET /users/dept/{deptId}`

## 八、线索管理

### 8.1 分页查询线索

`GET /clues`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| phone | String | - | 手机号 |
| status | Integer | - | 状态 1:待分配 2:待跟进 3:跟进中 4:伪线索 5:转商机 |
| channel | Integer | - | 来源 1:线上活动 2:推广介绍 |
| assignName | String | - | 归属人姓名 |
| page | Integer | 1 | 页码 |
| pageSize | Integer | 10 | 每页条数 |

### 8.2 新增线索

`POST /clues`

```json
{
  "phone": "13800138000",
  "channel": 1,
  "activityId": 1,
  "name": "张三",
  "gender": 1,
  "age": 25,
  "wechat": "zs123",
  "qq": "123456",
  "subject": 1,
  "level": 2,
  "remark": "来源双11活动"
}
```

### 8.3 根据ID查询线索

`GET /clues/{id}`

**响应 data 特殊说明：** 包含 `trackRecords` 列表（跟进历史）

### 8.4 分配线索

`PUT /clues/assign/{clueId}/{userId}`

**说明：** 将线索分配给指定销售，状态变为 2（待跟进）

### 8.5 跟进线索

`PUT /clues`

**请求体（Clue 对象）：**

```json
{
  "id": 1,
  "status": 3,
  "nextTime": "2026-06-01T10:00:00",
  "record": "客户对Java课程感兴趣，约了下周试听"
}
```

### 8.6 标记伪线索

`PUT /clues/false/{id}`

**请求体（ClueTrackRecord）：**

```json
{
  "falseReason": 1,
  "remark": "空号无法联系"
}
```

**说明：** falseReason 取值 1:空号 2:停机 3:竞品 4:无法联系 5:其他

### 8.7 转商机

`PUT /clues/toBusiness/{id}`

**说明：** 将线索转为商机，状态变为 5

### 8.8 线索池列表

`GET /clues/pool`

**说明：** 查询未分配或回收的线索，参数同分页查询

### 14.1 AI 跟进建议

**用途：** 销售填写跟进记录时，AI 自动分析历史记录，生成下一步行动建议

`POST /ai/followup-suggestion`

**请求体：**

```json
{
  "entityType": "business",
  "entityId": 1,
  "trackHistory": [
    {
      "record": "客户对Java课程感兴趣，询问了价格",
      "trackStatus": 1,
      "createTime": "2026-05-01 10:00:00"
    },
    {
      "record": "客户说需要跟家人商量",
      "trackStatus": 1,
      "createTime": "2026-05-05 14:00:00"
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| entityType | String | 是 | business / clue |
| entityId | Integer | 是 | 商机ID或线索ID |
| trackHistory | Array | 否 | 历史跟进记录，不传则自动查询 |

**响应 data：**

```json
{
  "suggestion": "客户已表达兴趣但需要决策时间，建议3天后再次联系，重点沟通课程优惠政策和试听体验",
  "suggestedNextTime": "2026-05-10T10:00:00",
  "keyPoints": ["重点沟通优惠政策", "邀请参加试听课", "了解客户顾虑"]
}
```

### 14.3 AI 线索评分

**用途：** 自动给线索打 A/B/C/D 等级，帮助销售优先跟进高意向客户

`POST /ai/lead-score`

**请求体：**

```json
{
  "clueId": 1
}
```

**响应 data：**

```json
{
  "score": "A",
  "confidence": 0.85,
  "analysis": "客户25岁，意向学科为AI智能应用开发，意向等级为'近期学习'，来源为线上活动，综合判断为高意向客户",
  "suggestedAction": "建议24小时内联系，重点介绍课程大纲和就业保障"
}
```

| 等级 | 说明 |
|------|------|
| A | 高意向，立即跟进 |
| B | 中等意向，近期跟进 |
| C | 低意向，保持联系 |
| D | 无意向，暂时搁置 |

## 十六、开发规范

### 16.1 请求方式对应

| 操作 | HTTP方法 | 说明 |
|------|----------|------|
| 新增 | POST | 添加一条记录 |
| 删除 | DELETE | 删除记录 |
| 修改 | PUT | 修改记录（全量） |
| 查询 | GET | 查询单条或列表 |

### 16.2 命名规范

| 后端 | 前端 | 说明 |
|------|------|------|
| listXxx | listXxx | 分页查询列表 |
