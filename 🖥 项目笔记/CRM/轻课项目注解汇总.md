# @DeleteMapping 超通俗讲解

## 1、是什么

`@DeleteMapping` 是 **SpringMVC / SpringBoot** 接口注解，

专门用来标注 **DELETE 请求方式** 的接口。

------

## 2、干什么用

HTTP 有几种常用请求方式：

- `@GetMapping`  → 查数据（GET）
- `@PostMapping` → 增数据（POST）
- `@PutMapping`  → 改数据（PUT）
- **`@DeleteMapping` → 删除数据（DELETE）**

@RequestParam 的三大核心作用

\1. **绑定 URL 查询参数（`?key=value` 形式）**

这是它最基础的作用：把 URL 里 `?` 后面的键值对参数，自动绑定到方法的形参上

## 2. **配置参数的「必传性」与「默认值」**

它可以直接控制参数是否必须传递，还能设置不传时的默认值，避免报错。

## 3**映射「参数名不一致」的场景**

当前端传的参数名和后端方法形参名不一样时，用 `@RequestParam("前端参数名")` 手动指定映射关系

## 一、PageHelper 作用

1. **MyBatis 分页插件**，专门帮你做**数据库分页**
2. **不用自己写 limit 分页 SQL**，自动拼接 `limit 起始,条数`
3. 自动计算：总条数、总页数、当前页、是否上下页
4. 配合 `PageInfo` 一键封装分页结果，直接返回前端

### 异常注解

@RestControllerAdvice 和 @ExceptionHandler 的f作用这两个注解用于全局异常处理，是Spring Boot中统一处理异常的机制。
1.@RestControllerAdvice
·作用范围:全局控制器增强注解
·功能:拦截所有 Controller 层抛出的异常
·特点:相当于@ControllerAdvice+@ResponseBody,返回JSON 格式数据
2. @ExceptionHandler
  ·作用:指定要捕获的异常类型
  ·位置:用在方法上
  ·功能:当发生指定异常时，执行该方法的处理逻辑
