## MoE 架构：DeepSeek vs LongCat-2.0

### 基本框架（两者相同）

```
输入 token → Router（门控网络）打分 → 选 top-k 专家 → 专家计算 → 加权合并输出
```

MoE = 一个路由院长 + 一群专家，每次只激活少数专家干活。DeepSeek R1/V3 和 LongCat-2.0 都是这套。

### LongCat-2.0 的三个工程创新

| 创新点 | DeepSeek | LongCat-2.0 |
|--------|----------|-------------|
| **专家分工** | 不预设，训练自动学会 | 预设三组：Agent / Reasoning / Interaction，按任务类型分组路由 |
| **激活参数** | 固定（如 37B/token） | 动态 33B~56B/token，简单 token 少激活 |
| **零计算专家** | 未激活专家仍占显存带宽 | 未激活专家连显存带宽都不占，真正零开销 |

### 关键参数

| | DeepSeek V3/R1 | LongCat-2.0 |
|--|---------------|-------------|
| 总参数 | 671B | 1.6T |
| 激活参数 | 37B | ~48B（动态） |
| 上下文 | 128K/1M | 原生 1M |
| 训练算力 | - | 五万卡国产集群 |
