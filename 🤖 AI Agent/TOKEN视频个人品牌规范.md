## 视频中的默认用法

人物形象默认只出现三次：

1. 钩子结束后的自我介绍。
2. 正文中一个关键讲解或观点迁移场景。
3. 视频最后的固定告别。

使用原则：

- 内容优先，人物不能全程占据画面。
- 人物不能遮挡字幕、标题或关键图表。
- 每个姿势必须使用独立透明 PNG，不能从包含多个人物的设定图中临时裁切。
- 预览时检查头发、手臂、手掌、手指和衣服边缘是否完整。
- 新视频可以调整人物坐标和大小，但不能改变已经确认的核心形象与服装。

## 默认品牌结构

```text
Hook → TokenBrandIntro → 正文 → 内容收尾 → TokenBrandOutro
```

- `TokenBrandIntro`：简短介绍 Token，并自然过渡到本期主题。
- 正文：人物只在一个真正需要强调的节点出现。
- `TokenBrandOutro`：固定告别与行动提示。
- 品牌版视频保留低透明度 `TOKEN_` 角标。

## 记录边界

以下内容属于具体视频实现，不写入长期个人品牌规范：

- 场景帧数与音频时间码
- 人物的 `top/right/left/bottom` 精确坐标
- 某一期视频使用的转场音与音量曲线
- 临时预览调整和故障排查过程

这些实现细节统一维护在对应项目的 `AGENTS.md` 和 `docs/` 中。
## 当前确认的视觉升级（2026-07-16）

从本次确认开始，TOKEN 视频和封面默认采用“工程解释现场”方向，而不是把通用科技 UI 作为主视觉。

### 视觉优先级

```text
真实问题冲突 > 工程因果关系 > TOKEN 人物表达 > 科技 UI 装饰
```

### 画面原则

- 优先呈现真实技术问题、调用链、终端状态、代码概念、错误标记和排查关系。
- TOKEN 人物是解释者、质疑者或排查者，人物服务于内容，不作为装饰贴图。
- 暗色背景、玻璃 UI、紫青高光和黄红强调色保留为辅助语言，不再主导整张画面。
- 整体更接近暗色工程现场、调试记录、技术白板和结构化草图。
- 避免通用赛博朋克、泛 AI 科技海报、过度霓虹、无意义粒子和装饰性面板。
- 人物不得遮挡字幕、标题、终端、卡片、关键图表或其他核心素材，也不得超出屏幕边界；空间不足时优先缩小人物。

### 封面原则

- 横屏封面：1920×1080；竖屏封面：1080×1920。
- 横屏和竖屏独立构图，不直接裁切。
- 封面提示词固定沿用“暗色工程现场 + 真实问题冲突 + TOKEN 人物解释 + 调用链/终端/代码状态 + 克制科技 UI”。
- 主题、核心冲突、人物动作、人物情绪、画面主体和封面标题由 Agent 根据最终脚本和成片自动提炼。
- AI 负责生成画面主体，中文标题默认后期叠加，确保文字准确。
