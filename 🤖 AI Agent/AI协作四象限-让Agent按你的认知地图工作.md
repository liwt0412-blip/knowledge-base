# AI 协作四象限：让 Agent 按你的认知地图工作

> **本质**：Johari 视窗（乔哈里视窗）× Agent 协作场景。把"你知道什么 / 你不知道什么"拆成四个格子，每个格子对应一种与 AI 协作的策略。
>
> 来源：晓辉博士分享的四象限方法论，结合 Loop Engineering 实操经验重述。

---

## 1. 四象限速览

```
              │  你知道             │  你不知道
──────────────┼────────────────────┼────────────────────
  AI 知道     │ ① 已知的已知       │ ③ 未知的已知
  (能执行)    │ 说清楚，全量传递   │ 出草稿→你判断→迭代
──────────────┼────────────────────┼────────────────────
  AI 不知道   │ ② 已知的未知       │ ④ 未知的未知
  (需探索)    │ 标注边界，让AI建议  │ 让AI主动照亮盲区
```

**一句话版本**：四象限告诉你什么时候"下指令"、什么时候"问问题"、什么时候"给反馈"、什么时候"让 AI 帮你发现你不知道的东西"。

---

## 2. 四象限详解 × 实操对照

### ① 已知的已知 —— 你能清晰描述，AI 能直接执行

**策略**：完整传递，不省略上下文。

**你的实操**：
- AGENTS.md 里写清楚背景、约束、技术栈、偏好
- spec 模板里把已知的部分结构化写出来
- 不假设 AI "应该知道"什么——默认它不知道，主动给

**反面案例**："帮我写一个登录功能" → AI 不知道你用 JWT 还是 Session、前端框架是什么、有没有现成的用户表。

**正向案例**："在 Spring Boot 3.2 + Vue3 项目中，用 JWT + Redis 实现登录，用户表是 `sys_user`，已有 `password` 和 `username` 字段，不要引入 Spring Security，保持轻量。"

### ② 已知的未知 —— 你知道自己不懂什么

**策略**：明确标注边界，让 AI 给出方案和取舍。

**你的实操**：
- 指令里标注 `[需AI建议]` 的部分
- "给目标不给步骤"——你定义要什么，让 CC explore 怎么做
- AI 给方案后，你判断取舍，不盲从

**反面案例**："前端用 React 还是 Vue？反正你看哪个好" → 放弃了自己的决策权，AI 没法替你决定团队偏好。

**正向案例**："这个模块技术栈我确定用 Spring Boot + MyBatis Plus。但消息队列选 RocketMQ 还是 RabbitMQ 我不确定 `[需AI建议]`，请给对比方案，考虑项目已有基础设施和团队经验。"

### ③ 未知的已知 —— 你判断好坏，但说不出标准

**策略**：AI 出草稿 → 你反馈 → AI 调整 → 规则沉淀。

**你的实操**：
- Loop Engineering 里的 self-review 环节——CC 出代码，你审，说不满意的地方
- 迭代几次后，把"为什么这个好那个不好"写成规则，丢进 AGENTS.md
- B端 UI 风格："#001529 sidebar / #E6F0FF 文本"——这就是从"我看得出来但说不清"沉淀出来的规则

**反面案例**："这个设计不好看" → 没有给出方向，AI 只能瞎猜下一版。

**正向案例**："这版太花哨了，回到上次的极简风格。蓝色渐变保留，但去掉阴影和圆角。"——每一次反馈都在把隐性标准变成显性规则。

### ④ 未知的未知 —— 你完全没意识到的盲区

**策略**：主动让 AI 扩展问题空间。

**你的实操**：
- 每个阶段结束时问 CC："有什么我可能忽略的问题？"
- Loop Engineering 里的"让 Agent 发现坑"——不给步骤，让 CC 自己走一遍，暴露你不知道的风险点
- 部署问题、边界条件、安全漏洞——这些做开发的人经常忽略，但 AI 可以从全局视角扫描

**反面案例**：写完代码直接上线，没问过 AI "这个接口有没有并发问题？"→ 上线后炸了才知道。

**正向案例**："方案我看了，在落地之前，请从部署、安全、性能、数据一致性四个角度扫描一遍潜在风险。"

## 3. 可复用的 Prompt 模板

以下模板可以**直接粘贴**给任何 AI Agent（CC、Hermes、ChatGPT、Cursor），按场景选用。

### 模板 A：任务开场的四象限声明

```
## 任务：[你的任务名]

### ① 我明确知道的
- [背景、约束、技术栈、已有代码]
- [偏好、风格要求]
- [不可更改的边界条件]

### ② 我知道自己不确定的 [需AI建议]
- [选项1 vs 选项2]：请给对比方案和取舍建议
- [不确定的技术点]：请分析可行性和风险

### ③ 我会通过反馈来沉淀标准
- 先出草稿/方案，我反馈后迭代
- 每一轮反馈都代表我的隐性偏好，请记录下来

### ④ 请帮我照亮盲区
- 在给出方案后，主动列出"[我可能忽略的问题]"
- 从安全、性能、部署、边界条件四个维度扫描
```

### 模板 B：方案审查的盲区扫描

```
请从以下维度扫描当前方案/代码的潜在问题：

1. 部署与运维：上线后会遇到什么坑？
2. 性能瓶颈：什么场景下会慢？
3. 安全漏洞：有没有我没意识到的攻击面？
4. 数据一致性：并发、分布式场景下会不会丢数据？
5. 边界条件：空值、超大值、网络断开、超时——这些情况处理了吗？

每个问题附严重程度（P0致命 / P1严重 / P2优化）和建议修复方案。
```

### 模板 C：隐性标准沉淀（迭代 3 轮后触发）

```
经过三轮迭代，请把我反复修改的点总结为规则：

1. 每次修改的"为什么"——我拒绝你初版的原因
2. 提炼成可复用的约束条件
3. 下一次同类任务可以直接遵守这些规则

输出格式：每条规则一行，用"必须/禁止/优先"开头。
```

## 4. 复利价值：把所有内容向"已知的已知"迁移

四象限的终极目标不是"每个象限都做好"，而是——**让 ②③④ 的内容不断向 ① 迁移**。

```
② 已知的未知  →  AI建议后你做出选择  →  变成 ① 已知的已知（决策已定，下次直接给）
③ 未知的已知  →  迭代反馈沉淀规则    →  变成 ① 已知的已知（审美标准已显式化）
④ 未知的未知  →  AI照亮后你意识到    →  变成 ① 已知的已知（盲区已被标注为检查项）
```

**具体怎么做**：每次协作后，把新发现的规则、标准、盲区**写回你的 AGENTS.md 或 spec 模板**。下一次 AI 自动继承——不需要重新教。

**这就是"AI 越来越懂你"的本质**——不是 AI 变聪明了，而是你把 ②③④ 里的模糊认知转化成了 ① 里的清晰指令。

---

## 5. 一句话使用指南

> 给任何 Agent 发任务前，花 10 秒走一遍四象限：
> ① 我知道的全部写出来 → ② 不确定的打标签 → ③ 等着出草稿给反馈 → ④ 最后问一句"有什么我没想到的"

### 本次验证出的制作规则

1. 钩子后使用固定的 4 秒通用标题场景，结构为 `Hook → TitleCard → 正文`。
2. 旁白使用 IndexTTS2 按场景设置情绪，先确认钩子试听，再批量生成。
3. 最终克隆音频是场景时长和字幕时间轴的唯一基准，字幕不能平均分配。
4. 转场音效保持统一风格，并使用更有冲击力的 whoosh/impact，但不得遮盖人声。
5. 视频规则写回项目 `AGENTS.md`，跨项目方法沉淀在 [[remotion科普视频]] 中。
