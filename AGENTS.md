# AGENTS.md - 小帅的知识库

## 知识库上下文（所有 Agent 适用）

知识库根目录：`D:\KnowledgeBase\小帅的知识库\`

### 启动与按需读取顺序

1. **必须先读**：`00-我的长期上下文.md` — 稳定协作规则与知识路由。
2. **根据任务读取**：长期上下文指向的一个模块入口 — 不默认加载其他模块。
3. **涉及目标或优先级时读**：`00-当前主线.md` — 当前唯一主线以此文件为准。
4. **找不到归属时读**：`00-知识库地图.md` — 全局目录与模块边界。

### 通用规则

- 执行任务时先读相关文件再判断，不要凭空补全项目事实。
- 修改知识库时优先归档，不要直接删除。
- 面试材料必须真实、抗追问，不要把没有直接负责的模块写成自己主导。
- 使用渐进式披露：先读模块入口，再读专题笔记，最后才读取原始资料或大文件。
- `AGENTS.md` 只定义所有 Agent 共用规则；Claude 专用 Hook 以 `CLAUDE.md` 和 `.claude/` 为准。

## 孤儿笔记处理（所有 Agent 通用）

发现孤儿笔记时采用分批、按需处理：

1. 先只查看候选文件的路径、标题和建议模块，不批量加载正文。
2. 每次最多处理 10 篇，并优先处理当前任务所属模块。
3. 选定文件后再读取完整内容，补充模块入口和 1～2 个语义相关链接。
4. 归档、备份、模板和 Agent 协议文件允许没有普通知识笔记的双向链接，不机械补链。
5. Claude 自动扫描与词汇提取的具体流程只在 `CLAUDE.md` 中维护。

### 文件处理

- 新增的 `.txt` 文件先转 GBK → UTF-8，再改 `.md` 扩展名。
- 新增的 `.md` 文件检查编码，GBK 的转 UTF-8。
- `!(path)` 图片语法转为 `![[path]]`。
