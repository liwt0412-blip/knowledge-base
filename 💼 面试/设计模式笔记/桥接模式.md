# 桥接模式（Bridge）

> 结构型设计模式

## 意图

将抽象部分与它的实现部分分离，使它们可以独立变化。解决多维度变化导致的类爆炸问题（M×N 个子类）。

## 两个维度的示例

| 维度 | 抽象 | 实现 |
|------|------|------|
| 数据库 | DriverManager | MySqlDriver / OracleDriver |
| 绘图 | Rectangle/Circle/Square | Red / Green / White |

## 代码示例

```java
// 数据库维度：抽象 = DriverManager，实现 = 具体Driver
public abstract class AbstractBridge {
    private Driver driver;
    public void setDriver(Driver driver) { this.driver = driver; }
    public Driver getDriver() { return driver; }
    public abstract void connect();
}

public class DriverManager extends AbstractBridge {
    public void connect() { getDriver().connect(); }
}

// 绘图维度：抽象 = Shape，实现 = Color
public abstract class AbstractShape {
    private Color color;
    public void setColor(Color color) { this.color = color; }
    public Color getColor() { return color; }
    abstract void draw();
}

public class Rectangle extends AbstractShape {
    public void draw() { getColor().painting("矩形"); }
}

// 测试：两个维度自由组合
// 数据库：切换驱动
DriverManager dm = new DriverManager();
dm.setDriver(new MySqlDriver("url", "user", "pwd")); dm.connect();
dm.setDriver(new OracleDriver()); dm.connect();

// 绘图：颜色 × 形状 自由组合
Rectangle rect = new Rectangle(); rect.setColor(new Red());   rect.draw();  // 红色矩形
Square    sq   = new Square();    sq.setColor(new Green());   sq.draw();    // 绿色正方形
Circle    cir  = new Circle();    cir.setColor(new White());  cir.draw();   // 白色圆形
```

## 使用场景

- 一个类存在两个（或多个）独立变化的维度
- 不希望用继承导致类爆炸（RedRectangle、GreenCircle... → M×N 子类）
- JDBC 是桥接模式的经典应用：DriverManager 是抽象，各厂商 Driver 是实现
