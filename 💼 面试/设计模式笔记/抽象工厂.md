# 抽象工厂（Abstract Factory）

> 创建型设计模式

## 意图

提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类。抽象工厂解决的是"产品族"的创建问题，确保同一产品族的对象能够协同工作。

## 结构

- **AComputerFactory** — 抽象工厂接口，定义 `createKeyBord()` 和 `createMouse()` 两个产品族方法
- **LianXiang01Factory / LianXiang02Factory** — 具体工厂，分别组装不同品牌组合的联想电脑
- **ProduceKeyBord / ProduceMouse** — 抽象产品接口
- **LeiBoKeyBord / LuoJiKeyBord / LeiBoMouse / LuoJiMouse** — 具体产品实现

## 代码示例

```java
// 抽象工厂接口
public interface AComputerFactory {
    ProduceKeyBord createKeyBord();
    ProduceMouse createMouse();
}

// 具体工厂01：键盘和鼠标都用雷柏品牌
public class LianXiang01Factory implements AComputerFactory {
    public ProduceKeyBord createKeyBord() { return new LeiBoKeyBord(); }
    public ProduceMouse createMouse()    { return new LeiBoMouse(); }
}

// 具体工厂02：键盘用雷柏，鼠标用罗技品牌
public class LianXiang02Factory implements AComputerFactory {
    public ProduceKeyBord createKeyBord() { return new LeiBoKeyBord(); }
    public ProduceMouse createMouse()    { return new LuoJiMouse(); }
}

// 测试
LianXiang01Factory factory01 = new LianXiang01Factory();
ProduceKeyBord kb1 = factory01.createKeyBord();
kb1.produceKeyBord("M550", "黑色");
ProduceMouse m1 = factory01.createMouse();
m1.produceMouse("M590", "有线");
```

## 使用场景

- 系统需要创建一系列相互关联的产品对象时
- 需要保证产品族的对象在组合使用时的一致性
- 横向扩展容易（新增产品族只需增加一个具体工厂），纵向扩展困难（增加新产品接口需要修改所有工厂）
