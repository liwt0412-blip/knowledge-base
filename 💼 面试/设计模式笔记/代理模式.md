# 代理模式（Proxy）

> 结构型设计模式

## 意图

为另一个对象提供替身或占位符以控制对这个对象的访问。三种代理：静态代理（编译期确定）、动态代理（运行时确定，Spring AOP 原理）、虚拟代理（延迟创建开销大的对象）。

## 三种代理对比

| 类型 | 绑定时机 | 灵活性 | 典型应用 |
|------|---------|--------|---------|
| 静态代理 | 编译期 | 低 | 简单场景 |
| 动态代理 | 运行时 | 高 | Spring AOP |
| 虚拟代理 | 按需 | 中 | 延迟加载 |

## 代码示例

```java
// 静态代理：编译时绑定
public class UserProxy implements CommodityService {
    private UserImpl user = new UserImpl();  // 预绑定
    public void getCommodity(String name) {
        System.out.println("我是UU跑腿，去帮用户取：" + name);
        user.getCommodity(name);
        System.out.println("商品已交给用户，期待好评");
    }
}

// 动态代理：运行时确定（Spring AOP 原理）
public class DynamicProxy implements InvocationHandler {
    private Object targetObj;
    public DynamicProxy(Object object) { this.targetObj = object; }
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("我是UU跑腿，去帮客户取：" + args[0]);
        Object result = method.invoke(targetObj, args);  // 反射调用
        System.out.println("商品已转交，期待好评");
        return result;
    }
}

// 工厂方法创建动态代理
public static Object getDynProxy(Object target) {
    return Proxy.newProxyInstance(
        target.getClass().getClassLoader(),
        target.getClass().getInterfaces(),
        new DynamicProxy(target));
}
```

## 代理 vs 装饰者

- **代理**：控制访问，被代理对象不对外暴露
- **装饰者**：添加行为，可多层嵌套

## 使用场景

- 静态代理：简单场景
- 动态代理：Spring AOP、MyBatis Mapper 代理
- 虚拟代理：延迟加载大对象
