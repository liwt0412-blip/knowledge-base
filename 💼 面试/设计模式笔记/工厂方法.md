# 工厂方法（Factory Method）

> 创建型设计模式

## 意图

定义一个创建对象的接口，但让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类，解决了直接 new 具体类带来的紧耦合问题。

## 结构

- **Chat** — 产品接口，定义聊天行为 `chatting(seqNo)`
- **QQ / WeiXin** — 具体产品，实现 Chat 接口
- **ChatFactory1** — 单方法工厂：通过参数判断返回不同产品（存在 null 风险）
- **ChatFactory2** — 多方法工厂：每种产品对应一个工厂方法
- **ChatFactory3** — 静态工厂：静态方法直接创建，无需实例化工厂

## 代码示例

```java
// 产品接口
public interface Chat {
    void chatting(String seqNo);
}

// 具体产品
public class QQ implements Chat {
    public void chatting(String seqNo) {
        System.out.println("使用QQ进行聊天 -- " + seqNo);
    }
}

// 静态工厂（推荐方式）
public class ChatFactory3 {
    public static Chat createQQChat()  { return new QQ(); }
    public static Chat createWeiXinChat() { return new WeiXin(); }
}

// 测试
Chat weixin = ChatFactory3.createWeiXinChat();
weixin.chatting("1");
Chat qq = ChatFactory3.createQQChat();
qq.chatting("3");
```

## 使用场景

- 当客户端不知道它所需要的对象的具体类时
- 当希望将对象的创建和使用解耦时
- 当系统需要灵活扩展新的产品类型时（开闭原则）
