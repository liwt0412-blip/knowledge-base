# 原型模式（Prototype）

> 创建型设计模式

## 意图

用原型实例指定创建对象的种类，并且通过拷贝这些原型来创建新的对象。核心是通过 `clone()` 方法复制已有对象，避免重复执行复杂的初始化过程。

## 浅拷贝 vs 深拷贝

- **浅拷贝**：基本类型和 String 被复制，但引用类型共享同一内存地址，修改会相互影响
- **深拷贝**：通过字节流序列化/反序列化，在内存中开辟全新的对象空间，修改互不影响

## 代码示例

```java
public class HomeWork implements Cloneable, Serializable {
    private Integer pages;
    private PupilStudent pupilStudent;  // 引用类型

    // 浅拷贝：引用类型共享同一对象
    @Override
    public HomeWork clone() throws CloneNotSupportedException {
        return (HomeWork) super.clone();
    }

    // 深拷贝：字节流序列化反序列化，完全独立
    public HomeWork deepClone() throws Exception {
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        new ObjectOutputStream(baos).writeObject(this);
        ByteArrayInputStream bais = new ByteArrayInputStream(baos.toByteArray());
        return (HomeWork) new ObjectInputStream(bais).readObject();
    }
}

// 浅拷贝问题：修改拷贝的学生信息，原对象也跟着变！
HomeWork myHomeWork = homeWork.clone();
myHomeWork.getPupilStudent().setName("Appleyk");  // 原对象也被改了！

// 深拷贝解决：完全独立
HomeWork deepCopy = homeWork.deepClone();
deepCopy.getPupilStudent().setName("张聪明"); // 原对象不受影响
```

## 使用场景

- 对象创建成本高（复杂数据库查询、网络请求），通过拷贝已有实例创建
- 需要避免重复初始化，对象间差异较小的场景
