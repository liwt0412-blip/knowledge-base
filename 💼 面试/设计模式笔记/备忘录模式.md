# 备忘录模式（Memento）

> 行为型设计模式

## 意图

在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态，以便日后可将该对象恢复到原先保存的状态。

## 结构

- **Original** — 原始类（发起人），`createMemento()` 保存状态，`restoreMemento()` 恢复
- **Memento** — 备忘录类，存储状态快照
- **Storage** — 管理者类，持有备忘录引用
- **OriginData / TransitionJournal / TransitionStorage** — 事务回滚版，支持按日志 ID 恢复

## 代码示例

```java
// 传统版：记住并恢复
Original original = new Original("明天周一，我们去吃火锅吧！");
Storage storage = new Storage(original.createMemento());
System.out.println("改变前：" + original.getValue()); // 吃火锅

original.setValue("明天周一，我们还是去吃北京烤鸭吧！");
System.out.println("改变后：" + original.getValue()); // 吃烤鸭

original.restoreMemento(storage.getMemento());
System.out.println("反悔后：" + original.getValue()); // 吃火锅（恢复！）

// 事务回滚版
List<Integer> data = new ArrayList<>(Arrays.asList(1, 2, 3));
OriginData originData = new OriginData(data);

TransitionStorage ts = new TransitionStorage(originData.createJournal()); // 日志1
originData.addData(4);
ts.backup(originData.createJournal());  // 日志2：增加后备份

originData.remove(2);
ts.backup(originData.createJournal());  // 日志3：删除后备份

originData.restoreJournal(ts.getLogMap(2)); // 恢复到日志2
System.out.println(originData); // [1, 2, 3, 4]  恢复成功！
```

## 使用场景

- 需要保存/恢复对象的状态快照（游戏存档、文本编辑器撤销）
- 事务回滚——保存操作前的数据状态
- Java 序列化、数据库事务日志、浏览器后退
