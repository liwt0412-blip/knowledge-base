# 命令模式（Command）

> 行为型设计模式

## 意图

将请求封装为对象，从而可用不同的请求对客户进行参数化，支持请求的排队、记录日志以及可撤销操作。将命令的发起者与执行者解耦。

## 结构

- **ICommand** — 命令接口，定义 `execute()`
- **ConcreteCommand** — 具体命令，持有 Receiver 引用
- **Invoker** — 调用者/发起者，持有命令对象
- **Receiver** — 接收者/执行者，真正执行命令逻辑

## 代码示例

```java
// 接收者：执行命令的人
Receiver receiver = new Receiver("吕蒙");

// 发起者持有一个具体命令（命令封装了接收者和指令内容）
Invoker leader = new Invoker(
    new ConcreteCommand(receiver,
        "去，趁关羽攻打许昌，赶紧偷渡长江把荆州给我抢回来！")
);
leader.setName("孙权");

// 发起者下达命令 → 命令对象 → 接收者执行
leader.action();
// 输出：
// 命令下达者：孙权
// 接收者：吕蒙，命令【去，趁关羽攻打许昌...】收到！
```

## 使用场景

- 需要将请求发送者和接收者解耦
- 需要支持撤销/重做操作
- GUI 按钮与后台逻辑、任务队列、宏命令录制
