# 模板方法（Template Method）

> 行为型设计模式

## 意图

定义一个操作中算法的骨架，将一些步骤延迟到子类中实现。使子类可以不改变算法结构即可重定义算法的某些特定步骤。

## 核心概念

- **模板方法**：`final` 修饰，定义固定算法步骤顺序
- **抽象方法**：子类必须实现的核心逻辑
- **钩子方法**：子类可选重写，提供扩展点

## 代码示例

```java
// 父类定义算法骨架
public abstract class AbstractCalculator {
    public final double calculate(String formula, String splitChar) {
        double[] arr = split(formula, splitChar);  // 步骤1：拆分（私有）
        hookSay();                                   // 步骤2：钩子（可重写）
        return calculate(arr[0], arr[1]);           // 步骤3：计算（抽象）
    }
    public void hookSay() {
        System.out.println("父类说：算法步骤的顺序我已经定好了！");
    }
    protected abstract double calculate(double a, double b);
}

// 子类只需实现核心计算
public class Plus extends AbstractCalculator {
    protected double calculate(double a, double b) { return a + b; }
    public void hookSay() { System.out.println("子类Plus说：由我来计算结果！"); }
}

// 使用
AbstractCalculator plus = new Plus();
System.out.println(plus.calculate("11 + 2", "\\+")); // 13.0
```

## 实际应用

- Spring JdbcTemplate：`execute(sql)` 是模板，`doInStatement(rs)` 留给子类
- HttpServlet：`service()` 是模板，`doGet()` / `doPost()` 留给子类
