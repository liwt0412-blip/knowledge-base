# 观察者模式（Observer）

> 行为型设计模式

## 意图

定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知并自动更新。又称发布/订阅模式。

## 结构

- **IObserver** — 观察者接口，定义 `discover()` 方法
- **INotify** — 通知接口，定义 `add()` / `remove()` / `notifyObservers()`
- **AbstractSubject** — 抽象主题，维护观察者列表
- **MySubject** — 具体主题，`extend()` 触发通知

## 代码示例

```java
// 标准版：发布/订阅
IObserver observerA = new ObserverA();
IObserver observerB = new ObserverB();
MySubject subject = new MySubject();
subject.add(observerA);
subject.add(observerB);
subject.extend(); // "大事不妙，有敌情！" → 通知所有观察者

// RSS 订阅版：支持退订
String content = "作者Appleyk发表了一篇博文";
CsdnSystem csdnSystem = new CsdnSystem(content);
csdnSystem.add(new RssUser(10001L, "曹操"));
csdnSystem.add(new RssUser(10002L, "刘备"));
csdnSystem.add(new RssUser(10003L, "孙权"));
csdnSystem.publish();   // 三人收到通知

csdnSystem.remove(10001L); // 曹操退订
csdnSystem.publish();   // 只有刘备、孙权收到
```

## 使用场景

- 一个对象改变需要同时通知多个对象
- 事件监听机制、消息推送、GUI 按钮点击事件
- Spring 事件机制、RxJava 响应式编程
