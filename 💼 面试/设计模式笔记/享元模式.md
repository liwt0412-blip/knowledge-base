# 享元模式（Flyweight）

> 结构型设计模式

## 意图

运用共享技术有效地支持大量细粒度对象的复用，减少对象创建数量，降低内存占用，提高性能。

## 结构

- **MyDbDriver** — 具体享元类
- **DbDriverFactory** — 享元工厂，内部维护 `HashMap`，先查缓存再新建
- **AbstractFlyWeight** — 抽象享元基类

## 代码示例

```java
DbDriverFactory factory = new DbDriverFactory();

MyDbDriver driver1 = factory.getDbDriver("mysql");     // 新建
MyDbDriver driver2 = factory.getDbDriver("mongodb");   // 新建
MyDbDriver driver3 = factory.getDbDriver("mysql");     // 复用！已存在
MyDbDriver driver4 = factory.getDbDriver("postgresql");
MyDbDriver driver5 = factory.getDbDriver("oracle");
MyDbDriver driver6 = factory.getDbDriver("postgresql"); // 复用！
MyDbDriver driver7 = factory.getDbDriver("mysql");      // 复用！

factory.showConns(); // 所有连接执行 connection()
System.out.println("工厂实例集大小：" + factory.size()); // 4，不是7！
```

## 使用场景

- 系统中有大量相似对象，造成内存开销过大
- 对象的大部分状态可以外部化（外部状态 vs 内部状态）
- 数据库连接池、线程池、String 常量池
