# 适配器模式（Adapter）

> 结构型设计模式

## 意图

将一个类的接口转换成客户端期望的另一个接口，使原本因接口不兼容而无法一起工作的类可以协同工作。

## 三种适配器

| 类型 | 方式 | 特点 |
|------|------|------|
| 类适配器 | extends 被适配者 + implements 目标接口 | 通过继承获得功能 |
| 对象适配器 | 持有被适配者引用 | 更灵活，可运行时替换 |
| 接口适配器 | 抽象类空实现所有方法 | 子类按需重写 |

## 代码示例

```java
// 类适配器：QQPlayer 继承 Mp3，直接拥有播放音乐功能
public class QQPlayer extends Mp3 implements Player {
    public void playMVs()    { System.out.println("QQ播放器实现播放MV功能"); }
    public void playMovies() { System.out.println("QQ播放器实现播放电影功能"); }
}

// 对象适配器：持有 Mp3 实例，可动态替换
public class BaoFengPlayer implements Player {
    private Mp3 mp3;
    public BaoFengPlayer(Mp3 mp3) { this.mp3 = mp3; }
    public void playMusics() { this.mp3.playMusics(); }
    public void playMVs()    { System.out.println("暴风影音播放MV"); }
    public void playMovies() { System.out.println("暴风影音播放电影"); }
}

// 测试
Player qqPlayer = new QQPlayer();
qqPlayer.playMusics();  // 继承自Mp3
qqPlayer.playMVs();

Mp3 sonyMp3 = new Mp3("索尼NW-ZX300A");
Player bfPlayer = new BaoFengPlayer(sonyMp3);
bfPlayer.playMusics();  // 委托给sonyMp3
```

## 使用场景

- 想使用已有类但接口不兼容
- 对象适配器比类适配器更灵活，Java 中优先使用
- Spring MVC 的 HandlerAdapter 就是适配器模式
