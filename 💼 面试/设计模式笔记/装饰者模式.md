# 装饰者模式（Decorator）

> 结构型设计模式

## 意图

动态地给对象添加额外的职责，比通过继承扩展功能更灵活。不修改原始类的前提下，为操作增加前置检查和后置通知。

## 结构

- **Folder** — 抽象构件接口，定义 `mkDir()` / `rmdir()`
- **FolderA / FolderB** — 具体构件，原始实现
- **FolderDecorator** — 装饰器，持有一个 Folder 引用，在调用原始方法前后添加额外行为

## 代码示例

```java
// 抽象构件
public interface Folder {
    void mkDir();
    void rmdir();
}

// 具体构件
public class FolderA implements Folder {
    public void mkDir() { System.out.println("在Windows系统中创建文件夹A"); }
    public void rmdir() { System.out.println("在Windows系统中删除文件夹A"); }
}

// 装饰器：在原始操作前后增加功能
public class FolderDecorator implements Folder {
    private Folder folder;
    public FolderDecorator(Folder folder) { this.folder = folder; }
    
    public void mkDir() {
        System.out.println("-- 检查文件夹名称是否含有非法字符 --");
        this.folder.mkDir();  // 委托给原始对象
        System.out.println("-- 提示用户文件夹是否创建成功 --");
    }
    public void rmdir() {
        System.out.println("-- 检查文件夹是否为空 --");
        this.folder.rmdir();
        System.out.println("-- 提示用户是否删除成功 --");
    }
}

// 测试：动态扩展功能，无需修改原始类
FolderA folderA = new FolderA();
FolderDecorator decorator = new FolderDecorator(folderA);
decorator.mkDir();  // 检查 → 创建 → 提示
```

## 使用场景

- 需要在不修改原始类的前提下动态添加功能
- Java I/O 的 `BufferedInputStream(new FileInputStream(...))` 是经典装饰链
- 遵循开闭原则：对扩展开放，对修改关闭
