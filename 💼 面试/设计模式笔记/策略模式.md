# 策略模式（Strategy）

> 行为型设计模式

## 意图

定义一系列算法，将每个算法封装起来并使它们可以相互替换。策略模式让算法的变化独立于使用算法的客户端。

## 结构

- **ICalculator** — 策略接口，定义 `calculate(formula)`
- **Plus / Sub / Mul** — 具体策略类（加、减、乘）
- **PriceContext** — 上下文类，持有策略引用，对外暴露 `discount()` 方法
- **Man300Jian100 / DianShang618 / Shuang11** — 具体打折策略

## 代码示例

```java
// 示例1：公式计算
ICalculator calculator = new Plus();
double result = calculator.calculate("2+ 8"); // 10.0

calculator = new Sub();
result = calculator.calculate("10 - 3");       // 7.0

// 示例2：商场打折——运行时切换策略
double totalPrice = 450.0;
PriceContext price = new PriceContext(new Man300Jian100());
System.out.println("折后价：" + price.discount(totalPrice)); // 350.0

// 切换策略：双11活动
price = new PriceContext(new Shuang11());
System.out.println("折后价：" + price.discount(1500.0)); // 随机折扣
```

## 使用场景

- 需要在运行时动态切换算法或行为
- 多个类仅在行为上不同，减少 if-else 分支
- 支付方式选择、排序算法切换、折扣计算
