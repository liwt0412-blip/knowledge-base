# 外观模式（Facade）

> 结构型设计模式

## 意图

为子系统中的一组接口提供一个统一的高层接口，使子系统更容易使用。外观层可以嵌套，逐层降低客户端复杂度。

## 结构

- **SubSystemA/B/C/D** — 子系统类（启动、加载数据、打招呼、工作）
- **FacadeA** — 封装模块A（启动 + 加载数据）
- **FacadeB** — 封装模块B（打招呼 + 工作）
- **FacadeAB** — 更高层封装（FacadeA + FacadeB）

## 代码示例

```java
// 子系统：模块A
public class SubSystemA {
    public void initSystem() { System.out.println("人工智能系统正在启动..."); }
}
public class SubSystemB {
    public void loadDatas() { System.out.println("正在加载数据..."); }
}

// 外观A：封装模块A的启动流程
public class FacadeA {
    private SubSystemA subSystemA = new SubSystemA();
    private SubSystemB subSystemB = new SubSystemB();
    public void initialize() {
        subSystemA.initSystem();
        subSystemB.loadDatas();
    }
}

// 外观AB：更上一层封装
public class FacadeAB {
    private FacadeA facadeA = new FacadeA();
    private FacadeB facadeB = new FacadeB();
    public void startSystem() {
        facadeA.initialize();
        facadeB.work();
    }
}

// 对比：不用外观 vs 用外观
// 不用外观：需了解4个子系统，手动调用每个步骤
SubSystemA a = new SubSystemA(); SubSystemB b = new SubSystemB();
a.initSystem(); b.loadDatas();

// 用外观：一步搞定
new FacadeA().initialize();      // 模块级
new FacadeAB().startSystem();    // 系统级，最简洁
```

## 使用场景

- 为复杂子系统提供简单入口
- 构建分层系统，每层定义外观入口点
- 子系统内部频繁变化，外观保持对外稳定接口
