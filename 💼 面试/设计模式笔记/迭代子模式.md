# 迭代子模式（Iterator）

> 行为型设计模式

## 意图

提供一种方法顺序访问聚合对象中的各个元素，而又不暴露该对象的内部表示。又称游标模式。

## 结构

- **IIterator** — 迭代器接口：`hasNext()` / `next()` / `first()`
- **AbstractAggregate** — 抽象聚集类：`createIterator()` / `size()` / `isEmpty()`
- **MyCollection** — 具体聚集类，底层用数组
- **MyIterator** — 具体迭代器，持有位置指针，`first()` 可重置

## 代码示例

```java
Integer[] numArr = new Integer[]{1, 2, 3};
AbstractAggregate aggregate = new MyCollection(numArr);
IIterator iterator = aggregate.createIterator();

// 第一次遍历
while (iterator.hasNext()) {
    System.out.println(iterator.next()); // 1, 2, 3
}

// 重置迭代器再遍历
System.out.println("first = " + iterator.first()); // 1
while (iterator.hasNext()) {
    System.out.println(iterator.next()); // 1, 2, 3
}

// 同一个聚集对象可创建多个互不干扰的迭代器
IIterator iteratorOther = aggregate.createIterator();
while (iteratorOther.hasNext()) {
    System.out.println(iteratorOther.next()); // 1, 2, 3
}
```

## 使用场景

- 需要遍历聚合对象但不想暴露其内部结构
- 支持多种遍历方式（正序、倒序）
- Java Collection 框架的 Iterator
