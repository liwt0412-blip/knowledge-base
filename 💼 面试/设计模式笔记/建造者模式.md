# 建造者模式（Builder）

> 创建型设计模式

## 意图

将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。客户端只需指定类型即可获得产品，无需关心内部构建细节。

## 结构

- **Cake** — 产品（Product）：最终要创建的复杂对象
- **CakeBuilder** — 抽象建造者：定义制作步骤接口 `knead()`、`ferment()`、`bake()`、`getCake()`
- **CakeMaker** — 具体建造者：由蛋糕师傅按工序制作
- **CakeSeller** — 指挥者（Director）：控制构建顺序，按固定流程组装产品

## 代码示例

```java
// 产品
public class Cake {
    private List<String> parts = new ArrayList<>();
    public void addProcess(String p) { parts.add(p); }
    public void show() { parts.forEach(System.out::println); }
}

// 抽象建造者
public interface CakeBuilder {
    void knead();           // 揉面
    void ferment();         // 发酵
    void bake(int minutes); // 烘烤
    Cake getCake();
}

// 指挥者：控制构建顺序
public class CakeSeller {
    public Cake sell(CakeBuilder builder) {
        builder.knead();    // 步骤1
        builder.ferment();  // 步骤2
        builder.bake(30);   // 步骤3
        return builder.getCake();
    }
}

// 客户端
CakeMaker cakeMaker = new CakeMaker("李师傅");
CakeSeller cakeSeller = new CakeSeller();
Cake cake = cakeSeller.sell(cakeMaker);  // 隐藏制作细节
```

## 使用场景

- 创建复杂对象，构建过程稳定但内部细节可变
- 构造步骤存在依赖顺序
- Lombok @Builder 是典型应用
