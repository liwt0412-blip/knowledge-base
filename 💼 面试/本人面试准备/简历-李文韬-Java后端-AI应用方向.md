## 项目经历

### 石化经营分析平台

**项目类型：** 省公司经营分析与业务管理平台  
**技术栈：** Spring Boot、Spring Cloud Alibaba、Nacos、Gateway、RabbitMQ、Redis、XXL-Job、MyBatis-Plus、MySQL、Spring AI、Milvus  
**项目背景：**  
该平台面向能源企业省公司、地市公司和油站一线人员，覆盖设备管理、经营台账、非油商品、客户核算、报表分析、告警通知和 AI 智能问答等场景。平台服务约 14 个地市、近 2000 座油站，目标是统一经营数据口径、提升设备异常处理效率，并降低一线人员查询制度和操作咨询的成本。

**个人职责：**

- 负责设备 IoT 接入与状态监控相关后端开发，处理设备上报数据消费、在线/离线状态判断、离线告警和通知链路。
- 参与 AI 助手省公司本地化落地，负责业务工具封装、知识库接入、RAG 检索链路联调、Redis 缓存、会话历史和 SSE 流式输出等关键工程能力。
- 配合非油、客户核算、网关通知等模块联调，排查权限、接口、SQL、缓存和数据问题，保障跨模块链路可用。
- 参与接口自测和前后端联调，使用 Apifox / Postman 验证接口参数、Token、权限边界和返回结构。

**核心实现：**

- **设备状态监控：** 设备上报数据进入 RabbitMQ 后异步消费，避免高峰期直接打满业务接口；消费端刷新 Redis TTL 心跳 Key，结合 XXL-Job 定时扫描判断设备离线状态。
- **离线告警处理：** 对未恢复告警做状态维护，避免每次扫描重复生成告警；设备恢复上报后更新设备状态并恢复告警。
- **AI 工具调用：** 将设备查询、告警查询、客户概览等 Java 业务能力封装为 Spring AI Tool，模型只负责选择工具和生成参数，后端负责参数校验、权限过滤和真实数据查询。
- **RAG 知识库接入：** Java 侧负责登录态、权限上下文、缓存、会话和业务入口；Python / RAG 侧负责文档解析、切分、Embedding、Milvus 检索和 Reranker 重排序。
- **成本与延迟控制：** 高频问题优先走 Redis 缓存，能查业务数据的问题走 Function Calling，无法通过工具解决的问题再进入 RAG 检索和大模型生成。
- **幻觉控制：** 回答基于内部文档或 Function Calling 返回的真实业务数据，RAG 回答附带引用来源，检索相关性不足时走兜底，不强行生成。

**项目收获：**

- 熟悉复杂企业后台系统中“业务权限 + 数据权限 + AI 工具调用”的集成方式。
- 对 RabbitMQ 削峰、Redis TTL 心跳、XXL-Job 补偿、接口幂等和告警去重有实际落地理解。
- 理解 AI 应用落地不是单纯接模型 API，而是要处理权限、缓存、会话、成本、延迟、稳定性和可观测性。
