# Java+AI 面试作战 MOC

> 上级入口：[[../面试作战首页|面试作战首页]]

## 使用原则

主线只有一条：

> Java 后端工程师 + 石化经营分析平台 + AI 助手本地化落地。

面试时先讲 Java 后端和石化项目，再根据面试官追问展开 RAG、Spring AI、Agent、微服务或补强项目。

不要一开始主动铺开所有 AI Agent、MCP、工具对比内容。

### 唯一主口径

同一个问题只保留一个主答案：

- 项目介绍、自我介绍：以“面试逐字稿”为准。
- 项目事实、职责和深挖证据：以“项目深挖追问底稿”的事实分级台账为准。
- AI 高频追问：以“AI 面试 30 问”为准。
- RAG 和 Spring AI 文档只补充深度，不得改写项目事实。
- 排错案例只有标记为“真实处理”或“参与联调”后，才能使用“我遇到过”。

冲突处理顺序：真实证据 > 项目底稿 > 逐字稿 > 专题答案 > 推演材料。

## 当前资料判断

当前作战手册适合的岗位：

- Java 后端开发。
- Java + AI 应用开发。
- 企业内部 AI 助手、知识库问答、经营分析、业务系统智能化岗位。

不适合直接对标的岗位：

- 大模型训练算法岗。
- 强算法竞赛岗。
- 纯基础架构内核岗。
- 要求完整 AI 平台从零设计的架构师岗。

当前核心优势：

- 有明确主项目，不是散点八股。
- AI 讲法能落到 Java 业务系统。
- 有真实 Bug、接口联调、团队协作材料。
- 有职责边界和红线，面试不容易讲过头。

当前主要短板：

- Java 基础仍然要高频刷。
- 项目真实指标不要乱编。
- AI 复杂概念只在追问时展开，不要主动堆术语。

最终面试目标：

> 让面试官相信你是一个能进项目、能写业务、能排问题、能把 AI 能力接进 Java 系统的后端开发。
