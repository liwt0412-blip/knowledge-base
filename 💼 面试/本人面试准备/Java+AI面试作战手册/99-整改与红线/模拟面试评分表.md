# 模拟面试评分表

## 使用原则

这张表不是为了安慰自己，是为了判断一个月后能不能稳定打 2W+ Java + AI 应用岗位。

评分标准：

- 每项 0-5 分。
- 总分 50 分。
- 40 分以上：可以主动投递 2W+ 岗位。
- 35-39 分：可以投，但要避开强算法、强底层、强大厂八股岗位。
- 30-34 分：需要继续补 Java 基础和项目追问。
- 30 分以下：不建议急着面核心岗位，容易被一轮打穿。

## 一、评分维度

| 维度 | 分数 | 5分标准 | 常见扣分点 |
|---|---:|---|---|
| 自我介绍 | 0-5 | 45 秒内说清 Java 后端、石化经营分析、AI 助手本地化落地 | 一上来讲太多 AI 概念；项目主线不清 |
| 项目概述 | 0-5 | 2 分钟讲清背景、业务目标、技术架构、自己职责 | 只说技术栈，不说业务价值 |
| 职责边界 | 0-5 | 能清楚区分总部平台、团队协作、个人实现范围 | 把所有 AI 底座都说成自己从零做 |
| AI 主链路 | 0-5 | 能讲清 Redis 缓存 -> Spring AI Function Calling -> Python RAG | 把 BERT/BM25/MCP 说成生产主线 |
| RAG 深挖 | 0-5 | 能讲清切分、召回、重排、上下文、幻觉控制、权限 | 只说“向量库查资料” |
| Java 微服务 | 0-5 | 能讲清 Spring Boot、事务、Redis、MQ、Seata 的项目使用边界 | 只背框架名，不知道适用场景 |
| Java 基础 | 0-5 | HashMap、线程池、JVM、Spring、MySQL、Redis 能稳定回答 | 基础概念断层，被追问就停 |
| 真实排错 | 0-5 | 能讲 2-3 个真实 Bug 的现象、定位、修复、复盘 | 只讲“我解决过很多问题” |
| AI 辅助开发 | 0-5 | 能讲 PRD/Spec、任务拆分、Agent 编码、人工验收、测试验证 | 只说“用 AI 提高效率” |
| 表达与风险控制 | 0-5 | 回答有边界、不夸张、不说绝对化词 | 说 100% 正确、彻底解决、全链路主导 |

## 三、2W+ 面试通过线

### 必须达到的水平

下面这些能力达不到，2W+ 岗位会比较危险：

- 能把石化经营分析平台讲成一个真实业务系统，而不是课程项目。
- 能说清自己负责了什么，没有负责什么。
- 能把 AI 助手讲成 Java 业务系统中的一个能力模块，而不是大模型研究项目。
- 能回答 Function Calling、RAG、Redis 缓存、权限、SSE、Python RAG 协作。
- 能回答 Java 基础 P0：集合、并发、JVM、Spring、MySQL、Redis。
- 能拿出真实 Bug 和接口联调案例，证明不是只会背。

### 允许存在的短板

这些短板可以接受，但要主动控制边界：

- 不需要把大模型训练讲得很深。
- 不需要把 LangChain、LlamaIndex、MCP 全部说成生产经验。
- 不需要把 Seata 说成自己深度改造过。
- 不需要在算法题上对标大厂强校招。

### 不能出现的风险

这些风险一旦出现，会明显掉分：

- 说 AI 底座全是自己从零搭建。
- 说 RAG 可以彻底解决幻觉。
- 说 Function Calling 100% 准确。
- 说 MCP 是石化项目当前生产主链路。
- 说具体提效百分比但没有统计依据。
- Java 基础只会说名词，不能解释原理和项目落点。

## 四、7天冲刺打法

### Day 1：主线表达

练：

- 自我介绍 45 秒。
- 石化项目 2 分钟。
- AI 模块 2 分钟。

验收：

- 不能超过时间。
- 不能主动铺开所有 AI 概念。
- 必须说清职责边界。

### Day 2：AI 主链路

练：

- Redis 缓存 -> Spring AI Function Calling -> Python RAG。
- RAG 为什么不能完全解决幻觉。
- Function Calling 怎么做参数校验和权限控制。

验收：

- 能回答 5 个连续追问。
- 每个回答都能回到业务场景。

### Day 3：Java 基础 P0

练：

- HashMap。
- ThreadLocal。
- 线程池。
- JVM OOM。
- Spring 事务失效。
- MySQL 索引。
- Redis 缓存问题。

验收：

- 每题 60 秒内能说清。
- 至少 70% 问题能讲出项目落点。

### Day 4：微服务与中间件

练：

- MQ 为什么异步。
- Redis 缓存一致性。
- Seata 适用边界。
- ES 和向量库区别。
- Spring Boot 自动配置。

验收：

- 不把所有中间件都说成自己深度负责。
- 能说出适用场景和不用它的原因。

### Day 5：真实开发案例

练：

- PageHelper 和 MyBatis Plus Page 冲突。
- LEFT JOIN 数据为空排查。
- ThreadLocal 当前用户上下文。
- 接口参数绑定错误。
- 权限字段必须后端控制。

验收：

- 每个案例讲清现象、定位、修复、复盘。
- 不讲成流水账。

### Day 6：AI 辅助开发

练：

- PRD/Spec 怎么写。
- Agent 怎么拆任务。
- AI 写完代码怎么验证。
- 怎么防止 AI 改坏代码。
- 怎么和人工 Code Review 配合。

验收：

- 不能只说“我用 AI 提效”。
- 必须说出测试、diff、自测、回归、边界验证。

### Day 7：压力模拟

练：

- 连续 45 分钟模拟面试。
- 每次回答后追问 2 层。
- 记录所有卡顿点。

验收：

- 总分达到 40。
- 没有明显夸大或逻辑冲突。
- 被问倒的问题能在当天补进对应文档。

## 五、面试复盘公式

每次面试后只复盘四件事：

1. 哪个问题没答出来？
2. 哪句话说过头了？
3. 哪个项目细节不够真实？
4. 哪个 Java 基础被打穿？

复盘后必须落到动作：

- 能补成逐字稿的，补到逐字稿。
- 能补成三层追问的，补到追问文档。
- 能补成红线的，补到整改 SPEC。
- 能补成真实案例的，补到 Bug 排查文档。

最终目标不是背完所有资料，而是形成稳定面试闭环：

> 主线清楚，边界准确，基础能扛，AI 有亮点，真实案例能证明经验。
