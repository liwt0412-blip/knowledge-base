# 面试题：Spring Cloud Gateway 路由与鉴权 — 2026.06.29

---

> 石化项目 7 个微服务里有一个网关鉴权服务。**Gateway 怎么把请求路由到后端微服务？鉴权过滤的具体流程——什么时候校验 Token、校验失败怎么处理？**

## 标准答案（40秒）

**路由转发：** Spring Cloud Gateway 从 Nacos 订阅所有微服务实例列表。前端请求到达 Gateway 后，先走 Predicate 断言匹配路由规则——比如 `Path=/api/non-oil/**` 匹配非油管理服务的请求。匹配后用 `lb://service-name` 从 Nacos 拿到对应服务的实例列表，轮询负载均衡选中一台转发。

**鉴权过滤：** GlobalFilter 对**所有路由**生效。先检查请求头有没有 Token，没有直接返回 401。有 Token 去验证——过期或签名错误也返回 401。**只有解析成功才放行**，把 userId 写入请求头转发到下游微服务。

**精髓：所有鉴权失败在网关层拦截，不穿透到业务服务。下游不用关心鉴权。**

---

## 结构速记

```
① Predicate 断言  → Path匹配 / Between时段限制
② Filter 过滤器链 → 鉴权(GlobalFilter) + 日志
③ lb:// → Nacos → 实例列表 → 轮询转发
④ 鉴权拒止分支   → 无Token→401 / 过期→401 / 解析失败→401
⑤ userId透传     → 写入请求头，下游直接用
```
