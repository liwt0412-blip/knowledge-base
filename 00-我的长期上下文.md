# 00-我的长期上下文

> 用途：这是所有 Agent 的轻量启动上下文。只保留稳定协作规则、个人主线与知识路由；具体知识必须按需从模块入口继续读取。

## 1. 当前主线

- 方向：Java 后端 / Java 微服务 / Java + AI 应用。
- 面试材料必须是根据项目来的,需要有项目感、可追问；明确区分直接负责、参与联调、了解架构和平台既有能力。
- 当前唯一主线是 Java + AI 方向的面试准备；具体优先级以 [[00-当前主线]] 为准。
- 天机学堂是当前学习项目之一，项目事实和学习进度从 [[🖥 项目笔记/00-项目模块入口]] 按需读取，不作为长期固定主线。

## 2. 协作方式

- 先读取相关文件和项目事实，再给出判断；重要改动先说明第一性原理、边界和取舍。
- 新增或修改代码时，为关键类、接口、业务分支和容易误解的字段补充中文注释；注释解释意图与约束，不逐行翻译代码。
- 编程时优先复用项目既有依赖、工具类、编码风格和 API 文档体系。
- 判断“项目规范”时不能只依据架构说明或通用习惯；必须同时检查公共机制的实现代码，并抽查多个现有业务模块的主流写法，再让新增代码与仓库实际惯例保持一致。
- 接入或改造已有接口时，先核对前端、网关、Feign 客户端和线上请求中的真实契约；仓库现有调用方是兼容性事实，新增枚举或字段设计不得直接破坏既有取值。
- 开始实施跨模块或涉及异步链路的功能前，先在项目仓库编写并确认 Spec；Spec 至少说明范围、接口、数据模型、事件/一致性边界、失败补偿与验收条件，再进入编码。
- 处理长期知识时不把聊天记录、临时日志、一次性报错塞入本文件；先归档到对应主题笔记。

## 4. 记忆写入规则

- 用户说“加入知识库”：归档到最具体的主题笔记；必要时更新对应模块入口。
- 用户说“加入长期上下文”：只写未来会反复影响决策的稳定规则；项目细节、接口字段、日志和一次性排错仍写专题笔记。
- 如果归属不清，先写入可追溯的专题笔记并标记待整理，不把不确定结论写入长期上下文。
- 长对话或阶段任务完成后：保留“已做决策、当前状态、未解决问题、文件入口”，压缩过程性对话与原始工具输出。

## 5. 读取顺序

```text
长期上下文 → 当前任务所属模块入口 → 专题笔记 / 项目文件 → 当前任务
```

不要默认全量读取知识库。
