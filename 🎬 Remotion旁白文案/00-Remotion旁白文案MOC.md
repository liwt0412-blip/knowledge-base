## 原始素材转旁白

创建或改写旁白前，必须按顺序完整读取：

1. [[🤖 AI Agent/Hermes Agent技能库/token-人味儿写作]]

默认结构为“共鸣 → 问题 → 方案 → 原理 → 限制/判断 → 具体结论”。不编造 TOKEN 的经历，不从定义或宏大背景开始，不使用宣传腔和万能展望。原创稿全部是本次新增内容，因此事后检查覆盖整篇。

原始素材整理成可审核旁白后，将文案保存到本目录并设置 `status: 待审核`。用户确认前不进入正式配音。
## 标准状态流转

```text
待创作 / 待审核
→ 已确认
→ 配音中
→ 已配音
→ 制作中
→ 待预览
→ 待渲染
→ 待发布
→ 已完成
```

| 状态 | 说明 |
|---|---|
| 待创作 | 已有选题或素材，尚未形成可审核旁白 |
| 待审核 | 已有旁白草稿，等待用户确认 |
| 已确认 | 脚本已锁定，可以生成正式旁白 |
| 配音中 | 正在生成或确认克隆旁白 |
| 已配音 | 最终旁白确定，准备强制对齐与时间轴制作 |
| 制作中 | 正在实现字幕、画面、动画和音效 |
| 待预览 | 代码检查通过，等待 Studio 预览确认 |
| 待渲染 | 预览已确认，等待用户明确授权渲染 |
| 待发布 | MP4 已完成，等待封面与发布文案 |
| 已完成 | 成片、发布物料、路径回写和用户终验全部完成 |

`template`、`active` 和 `已完成` 不进入创作队列。

## “已完成”的判断

必须同时具备：

- MP4 已成功渲染且用户确认成片。
- 横屏和竖屏封面已完成；明确放弃的交付物已记录。
- 标题候选、发布正文、话题标签和首评互动引导已保存。
- 原文案 YAML 和“制作产物”区域已经回写最终路径。
- 用户确认本期可以结束。

缺少任一项就保留真实阶段，不提前标记完成。

## 队列选择规则

1. 只读取本目录，不扫描整个知识库。
2. 优先级按 `high → normal → low`。
3. 同优先级按 `updated` 或 `date` 从早到晚。
4. 多个候选无法唯一确定时先展示给用户选择。
5. 选中后读取其 `source` 来源笔记，核对事实与上下文。

## 生产规则

1. 每个视频单独建立一份文案，不覆盖旧版本。
2. 旁白先确认，再生成正式配音。
3. 最终音频决定场景时长、字幕时间轴和动画触发点。
4. “预览”只表示 Remotion Studio，不授权渲染 MP4。
5. 状态变化时更新 YAML `status`、`updated` 和 `next_action`。
6. 暂停或失败时记录 `blocked_reason`，不伪装成完成。
