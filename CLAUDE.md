# CLAUDE.md - 小帅的知识库

## 通用知识库规则

开始工作前必须先读取并遵守根目录 `AGENTS.md`。身份上下文、渐进式读取顺序和面试真实性红线均以 `AGENTS.md` 为统一规则源；本文件只补充 Claude 专用 Hook 行为。

## 每次会话开始时

### 孤儿文件链接（由 scan-orphans Hook 自动触发）

会话启动时若看到 `ORPHAN FILES DETECTED` 标记，说明存在缺少 wikilink 的候选文件。Hook 只输出轻量候选清单，不代表必须在一次会话全部处理。

1. 每次最多处理候选清单中的 10 篇，优先选择当前任务所属模块。
2. 选定文件后再读取完整内容，提取关键主题词。
3. 用 Grep 在 vault 中搜索这些关键词，找到内容相关的笔记。
4. 在孤儿文件中添加或更新“相关笔记”区域：
   - 模块入口回溯链接（必须）；
   - 语义相关的笔记链接（至少 1～2 个）。
5. 在目标笔记中也添加双向链接（如果目标有相关笔记区域）。
6. 处理完成后，将本批文件路径写入 `.claude/hooks/vocab-queue.txt`。
7. 写入后运行 `python ".claude/hooks/extract-vocab.py"` 触发词汇提取。

归档、历史备份、模板和 Agent 协议文件可以保留为非普通知识节点，不为了消除孤儿数量机械补链。

### 词汇提取（由 extract-vocab Hook 自动触发）

会话启动时若看到 `VOCAB EXTRACTION REQUIRED` 标记，说明有新文件排队等待词汇提取。

1. 从候选词中去重（已存在于词汇大全中的跳过）。
2. 对每个新词汇提供：IPA 音标、词根拆解、中文含义、来源文件。
3. 格式沿用英语词汇大全现有表格风格。
4. 根据词汇所属领域放入对应主题区。
5. 写入完成后清空 `.claude/hooks/vocab-queue.txt`。

### 文件处理

文件编码和图片语法遵循根目录 `AGENTS.md`，不在本文件重复维护第二套规则。
