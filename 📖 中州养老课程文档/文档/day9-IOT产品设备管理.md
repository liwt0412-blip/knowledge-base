Day11 - IOT产品设备管理



1- 智能监测模块介绍

	1- 作用：通过 【智能硬件设备感知】 + 【智能设备数据传输上报】 的形式 对生活各个场景进行监控、识别、报警，

		     达到智能辅助、缩减人力成本、增加便捷性和安全性 的目的

	2- 举例：老人入住养老院之后，可以通过选择配置来配备智能硬件设备，在两类生活场景发挥重要作用：

		- 安全场景：防跌倒意外、防走失、睡眠中意外、区域滞留、环境安全

			- 烟雾报警器：监控火灾、烟雾

			- 睡眠检测带：监控睡眠质量、睡眠意外

			- 门磁报警器：监控房间出入

			- 跌倒报警器：监控跌倒

			- 滞留报警器：监控卫生间滞留

		- 健康场景：健康监测、健康小屋；

			- 健康手表：监控各项健康指标、定位

	3- 其中关于【智能设备数据传输上报】采用 【华为云 IOT平台】提供的设备接入服务（IoTDA）

	4- 目标：

		1- 熟悉华为云IoTDA平台的特点

		2- 掌握华为云IoTDA平台的一些概念，如：【产品、设备、物模型】等

		3- 使用华为云IoTDA平台提供的接口创建设备，并绑定业务数据（老人、位置）

		4- 熟悉智能设备对接华为云IoTDA平台上报数据的方式

	1- 物联网介绍

		1- 概念：把所有物品通过【信息传感设备】，按约定的协议，与【互联网】连接起来，进行【信息交换】，即【万物互联】

		        以实现智能化识别和管理的目的，如：智能化识别、定位、跟踪、监管等功能

		2- 应用场景：共享充电宝、智能音箱、智能家居、智能农耕、智能医疗

3- 华为IOT平台-创建产品&物模型&设备

	1- 产品&物模型&设备概念

		0- 官方参考：https://support.huaweicloud.com/productdesc-iothub/iot_04_0011.html

		1- 产品：某一类具有相同能力或特征的设备的集合称为一款产品，比如 手表、睡眠检测带、烟雾报警器

		2- 物模型 -产品模型

			- 由服务、属性、命令三部分构成，用于描述产品具备的能力和特性，能使IOT平台 能理解该款设备支持的服务、属性、命令

			- 服务：即Service，产品模型的一部分，描述设备具备的业务能力。将设备业务能力拆分成若干个服务后，再定义每个服务具备的属性、命令以及命令的参数。

			- 命令：设备的功能模型之一，设备能够被外部调用的能力或方法。

			- 属性：即Property，产品模型的一部分，一般用于描述设备运行时的状态，如环境监测设备所读取的当前环境温度等。

				- 如 手表 产品属性电量、身体血压、血氧、体温数据

		3- 设备：归属于某个产品下的设备实体，每个设备具有一个唯一的标识码。设备是绑定在产品上的，所以必须先创建产品才行

				 设备可以是直连物联网平台的设备，也可以是代理子设备连接物联网平台的网关。

	2- Java SDK使用示例

		1- 文档路径

			直达路径：https://support.huaweicloud.com/sdkreference-iothub/iot_02_0178.html

		2- 介绍

			1- 如果是自己写代码，需要导入的依赖

				1- 介绍项目下每个模块的作用

				2- 我们需要的模块：iot-device-demo：普通直连设备的demo代码。

			3- 结合在线讲义解读 iot-device-demo 模块下的 PropertySample 类，核心：

				1- 指定 设备接入上报的地址：控制台 - 右上角 - 接入信息 - MQTTS 接入地址 2a2cb2fbc0.st1.iotda-device.cn-north-4.myhuaweicloud.com

				2- 指定 设备ID、设备密钥

				3- 指定 上报物模型服务名、物模型数据

		3- 解压打开项目
