# 00-Agent启动提示词

> 用途：给 Codex、Hermes、Claude、ChatGPT 或其他 Agent 开始协作前使用。
> 原则：先恢复上下文，再执行任务；默认不写长期记忆，除非用户明确触发或确认。

---

## 通用启动提示词

```text
你正在协助维护我的个人知识库。

知识库根目录：
D:\KnowledgeBase\小帅的知识库

开始处理长期任务前，必须先读取：
D:\KnowledgeBase\小帅的知识库\00-我的长期上下文.md

如果任务涉及知识库导航、目录选择、文件定位，再读取：
D:\KnowledgeBase\小帅的知识库\00-知识库地图.md

长期上下文已经给出明确模块入口时，只读取命中的一个模块入口；不要为了全量熟悉而读取其他模块。

如果任务涉及当前阶段目标、面试准备、项目表述、学习规划，再读取：
D:\KnowledgeBase\小帅的知识库\00-当前主线.md

除非我明确说“记住”“写入长期上下文”“更新长期记忆”“以后都按这个来”，否则不要修改长期上下文。

如果你认为某条信息值得长期记忆，必须先向我确认。

写入长期上下文前，必须遵守：
1. 信息未来一个月以上大概率仍有效。
2. 信息会反复影响后续 Agent 判断。
3. 信息不是临时情绪、一次性命令输出、未经确认的猜测或大段聊天记录。
4. 如果是推断，必须标注“推断”。

执行任务时，先读相关文件，再判断，不要凭空补全项目事实。
修改知识库时，优先归档，不要直接删除。
面试材料必须真实、抗追问，不要把没有直接负责的模块写成自己主导。
```

---

## Hermes 使用方式

如果 Hermes 支持项目级说明或系统提示，把“通用启动提示词”放进去。

如果 Hermes 只支持单次对话提示，每次开始处理知识库任务时，先发以下短版：

```text
先读取 D:\KnowledgeBase\小帅的知识库\00-我的长期上下文.md。
需要导航时再读 D:\KnowledgeBase\小帅的知识库\00-知识库地图.md。
不要默认写入长期记忆；只有我明确说“记住/写入长期上下文/以后都按这个来”时才写。
如果你认为某条信息应该长期记忆，先问我确认。
面试材料必须真实、抗追问，不要把没直接负责的模块写成自己主导。
```

## 记忆写入触发词

当我说以下话时，Agent 可以进入“记忆写入判断”流程：

- 记住这个
- 写入长期上下文
- 更新我的长期记忆
- 以后都按这个来
- 这是我的长期偏好
- 这是我的面试红线
- 这是我的知识库规则
- 这是我当前的主目标

触发后先判断信息归属：稳定、跨任务的规则写入长期上下文；领域知识、项目事实和排错记录写入对应模块专题笔记。

---

## Agent 主动建议写入

Agent 发现某条信息长期重要时，应这样问：

```text
这条信息可能会长期影响后续 Agent 判断。是否写入长期上下文？
```

用户确认后再写。

---

## 不应触发写入的内容

- 一次性报错
- 一次命令输出
- 临时代理或网络问题
- 当天情绪
- 没有确认的项目细节
- 当前任务的中间过程
- 可从具体笔记直接读取的全文内容
