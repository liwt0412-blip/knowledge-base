# AI使用心得：先找现成，不要手搓

晓辉博士分享的反直觉AI使用心得：**当你想让AI帮你实现某个功能时，不要直接让它"手搓"应用，而是先让它去GitHub上搜索是否已有现成的开源项目，这样更高效。**

---

## 1. 具体例子

她的需求：在浏览网页或PDF时，希望能"指哪儿打哪儿"地解释不明白的句子或词汇，同时能保存对话到知识库。

### 以前的做法

复制粘贴到AI聊天工具 → 脱离上下文、麻烦。

### 现在的做法

让AI去GitHub上搜索相关功能 → 找到现成的**Chrome插件** → 直接安装使用 → 在现有项目基础上让AI补充完善。

---

## 2. 为什么不手搓更好

| 原因 | 说明 |
|------|------|
| **节省Token和成本** | 不需要让AI从头生成大量代码 |
| **节省时间** | 直接可用，不需要调试开发 |
| **避免从零开发的Bug** | 成熟项目已经过验证和修复 |
| **站在巨人肩膀上** | 利用已验证的成熟方案，起点更高 |

---

## 3. 适用场景

这种思路不仅适用于浏览器插件，很多场景都可以用：

- 搭建PPT工作台
- 自动化工作流
- 数据处理工具
- 任何"I want X to do Y"的需求

**核心原则：先搜索有没有现成的，再让AI在成熟方案上修改完善。**

---

## 4. 工作方式转变

- 这是一种**工作方式和习惯的改变**，不是技术问题
- 普通人也应该适应这种与AI的协作方式
- 如果你在某个领域很专业，也可以把自己的成果**开源到GitHub上**，让别人站在你的肩膀上
