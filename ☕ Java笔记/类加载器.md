# **类加载器、类加载机制、加载全过程**

## 一、什么是类加载器（ClassLoader）？

**Java 虚拟机不会一开始就把所有类加载进内存**。

当运行程序，第一次用到某个类（`new`、调用静态方法、反射等）时，**负责把 `.class` 文件从硬盘读到内存，并生成一个 Class 对象的工具 → 就叫 类加载器**。

简单比喻：

- **.class 文件** = 课本（在硬盘里）
- **JVM 内存** = 大脑
- **类加载器 ClassLoader** = **把课本翻开放进大脑的人**
- 

## 二、Java 内置三大类加载器

从上到下**层级关系（双亲委派）**：

1. **Bootstrap ClassLoader 启动类加载器**
- C++ 实现，JVM 自带
   - 加载 **JRE/lib** 下的核心类（`String、Object、HashMap…`）
- 最顶级，无父加载器
   
2. **Extension ClassLoader 扩展类加载器**
- Java 实现
- 加载 **JRE/lib/ext** 下的扩展类
  

3. **Application ClassLoader 应用类加载器（系统类加载器）**

   - 加载 **我们自己写的类、项目里的类**
- 平时 `main` 方法所在的类就是它加载的
   
   

## 三、双亲委派模型（Parent Delegation）

### 规则：

1. 类加载器收到加载请求
2. **自己不先加载，先交给父加载器**
3. 父也交给更上层
4. 一直传到**启动类加载器**
5. 上层能加载就加载，不能加载才交给下层

### 目的：

**保证核心类（如 String）不会被你自己写的假 String 类替换！**

保证安全、保证类全局唯一、不重复加载。



## 四、类的完整加载过程：加载 → 连接 → 初始化

### 1. 加载（Loading）

- 找到 `.class` 文件
- 读二进制数据
- 在内存中**创建一个 Class 对象**（方法区）

### 2. 连接（Linking）分 3 小步

#### ① 验证（Verify）

检查 class 文件是否合法、是否符合 JVM 规范

#### ② 准备（Prepare）

- **为静态变量分配内存**

- 赋予默认初始值

  （int=0、boolean=false、引用 = null）

  注意：这里只是赋默认值，不是你代码里写的值！

#### ③ 解析（Resolution）

把符号引用变成直接引用（把类名变成真实内存地址）

### 3. 初始化（Initialization）【真正执行代码】

- 执行 **静态代码块 static {}**
- 给**静态变量赋真正的值**



## 五、什么时候会触发类初始化？

只要出现下面任意一种，就会触发类加载 + 初始化：

1. **new 对象**
2. 访问**静态变量 / 静态方法**
3. 反射调用 `Class.forName()`
4. 初始化子类，父类先初始化
5. 执行 `main` 方法的主类



## 六、小结

1. **类加载器：把 .class 读进内存，生成 Class 对象**
2. **三大加载器：启动 → 扩展 → 应用**
3. **双亲委派：优先交给父加载器，安全唯一**
4. **类加载过程：加载 → 验证 → 准备 → 解析 → 初始化**
5. **准备：赋默认值；初始化：赋真正值**





## 七、代码逐行讲解

```java
class Test {
    // 静态变量
    static int a = 10;
    
    // 静态代码块
    static {
        System.out.println("静态代码块执行了");
        a = 20;
    }
}

public class Main {
    public static void main(String[] args) {
        System.out.println("main开始");
        Test t = new Test(); // 第一次使用Test类，触发类加载
        System.out.println(Test.a);
    }
}

// 运行结果：
// main开始
// 静态代码块执行了
// 20
```



### 第一步：加载（Loading）

- 找到 `Test.class` 文件
- 把**二进制数据读进内存**
- 在**方法区**创建唯一的 **Class<Test> 对象**
- **此时：还没给变量赋值，也没执行代码**



### 第二步：连接（Linking）—— 3 小步

#### 1. 验证（Verify）

检查 `.class` 是否合法、有没有被篡改、是否符合 JVM 规范。

#### 2. 准备（Prepare）

- **只为静态变量 (static) 分配内存**

- 只赋默认值

  - `int a` → **0**
  - `引用类型` → **null**
  - `boolean` → **false**


此时：`a = 0`（不是 10！不是 20！）

#### 3. 解析（Resolution）

把代码里的 `Test` 变成**内存地址**，方便 JVM 查找。



### 第三步：初始化（Initialization）

**只有这里才执行你写的代码！**

执行顺序**固定死**：

1. 静态变量显式赋值：`static int a = 10`
2. 静态代码块：`static { ... }`

所以：

- 先：`a = 10`
- 再：`a = 20`
- 最终：`a = 20`



### 问：什么时候才会触发上面整个流程？

**答：只有第一次真正使用类时才加载！**

触发条件：

1. **new 对象**
2. 访问**静态变量**
3. 访问**静态方法**
4. **反射** `Class.forName("Test")`
5. 初始化子类，父类先加载



## 八、加载过程小结

1. **加载**：把 class 文件读进内存，生成 Class 对象
2. **验证**：检查文件是否合法
3. **准备**：静态变量分配内存，**赋默认值 0/false/null**
4. **解析**：符号变地址
5. **初始化**：执行**静态代码 + 静态赋值**
