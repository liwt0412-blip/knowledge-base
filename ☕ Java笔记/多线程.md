## 进程介绍

正在内存中运行的应用程序

进程 (Process) 是计算机中的程序关于某数据集合上的一次运行活动---正在运行的应用程序

是系统进行资源分配的基本单位

1.独立性：每一个进程都有自己的空间，在没有经过进程本身允许的情况下，一个进程不可以直接访问其它的的进程空间

2.动态性：进程是动态产生，动态消亡的

3.并发性：任何进程都可以同其它进程一起并发执行

### **并行和并发**

l并行：在同一时刻，有多个指令在多个CPU上【同时】执行 

l并发：在同一时刻，有多个指令在单个CPU上【交替】执行

选择并行还是并发,要看cpu的运行情况如果有空闲的就能并行,如果没有空闲的就只能并发

多进程同时工作:

 对于一个 CPU 而言，它是在多个进程间轮换执行的,多核的就可以并行

### **线程介绍**

进行中,一个一个的线程任务

进程中,负责执行代码的一个一个的执行单元

l线程 (Thread) : 进程可以同时执行多个任务，每个任务就是线程

### **多线程的意义**

l随着处理器上的核心数量越来越多，现在大多数计算机都比以往更加擅长并行计算

l一个线程，在一个时刻，只能运行在一个处理器核心上

提高执行效率

同时处理多个任务

### java开启线程的方式

继承Thread类

实现 Runnable 接口  ---线程任务无返回值

实现 Callable 接口 ----线程任务有返回值

start()  开启线程

### 线程池

将线程对象交给线程池维护,可以降低系统成本,从而提升成程序的性能

系统创建一个线程的成本是比较高的,因为它涉及到与操作系统交互

当程序中需要创建大量生存期限很短暂的线程时,频繁的创建和销毁线程,就会严重浪费系统资源
