# 全局异常处理器

## 一、代码实现：

```java
@Slf4j  //  记录日志，目的是输出错误信息
@RestControllerAdvice  //  标识这是一个全局异常处理器
public class GlobalExceptionHandler{
	@ExceptionHandler  //  标识当前是一个异常处理的方法
	public Result handleException(Exception e){
        // 输出错误信息
        log.error("出错程序：", e);
        // 返回一个错误的响应结果
		return Result.error("服务器异常，请稍后重试");
	}
}
```



## 二、异常捕获原则

从小到大，匹配最近的异常。

例如，DuplicateKeyException这个异常，它的“祖宗链”就是：DuplicateKeyException ==> DataIntegrityViolationException ==> NonTransientDataAccessException ==> NestedRuntimeException ==> RuntimeException  ==> Exception  ==>  Trowable  ==> Serializable。

假如现在程序抛出一个 NestedRuntimeException异常，此时程序会先从DuplicateKeyException 开始，一级一级往上找，知道找到最能匹配的那个异常。

注意：一个全局异常处理器中可以书写多个不同类型的异常，**且异常的捕获与书写先后顺序无关**。



## 三、异常处理的5个核心关键字

- **`try`**：用来包裹**可能会出问题**的代码块。
- **`catch`**：用来**捕获并处理** `try` 中发生的特定异常。如果没发生异常，`catch` 里的代码不会执行。
- **`finally`**：**无论是否发生异常，这里的代码都一定会执行**。通常用来做“收尾工作”，比如关闭文件、断开数据库连接等，防止资源泄露。
- **`throw`**：用在方法内部，**手动抛出一个具体的异常对象**。比如你发现用户输入的年龄是负数，就可以手动 `throw new IllegalArgumentException("年龄不能为负")`。
- **`throws`**：用在**方法的声明上**，告诉调用者：“我这个方法可能会抛出某些异常，你要做好准备（要么捕获，要么继续向上抛）”。



## 四、常用异常类API

| 异常API                   | 描述                                           |
| ------------------------- | ---------------------------------------------- |
| **`e.getMessage()`**      | 获取异常的详细描述信息（字符串形式）           |
| **`e.printStackTrace()`** | 直接在控制台打印出完整的错误堆栈信息（最常用） |
| **`e.toString()`**        | 返回异常的简短描述（包含异常类名和消息）       |



## 五、利用异常API定制报错输出信息

例如，当带有唯一约束的字段重复时，会动态报错，例如“123456已存在”。

```java
public Result handleDuplicateKeyException(DuplicateKey){

    String errMsg = message.substring(message.indexOf("Duplicate entry"));
    // 将截取的子串根据空格分割，分割后第3个元素就是重复的字段名

}
```



## 六、自定义异常

**目的**：为了让控制台的报错信息更加见面之意。

**自定义过程：**

1. 定义异常类：异常类名要见名知意
2. 写继承关系：如果定义的是运行时异常，要继承RuntimeException，如果是编译时异常，就直接继承Exception就可以了
3. 添加空参构造和带参构造

**代码示例：**

```java
Public class BusinessException extends RuntimeException(){
    public BusinessException(){}
    public BUsinessException(String message){super(message);}
}
```

运行时：RuntimeException 核心 就标识由于参数错误而导致的问题

编译时：Exception 核心 提醒程序员检查本地信息



## 七、**进阶小技巧：自动释放资源 (try-with-resources)**

如果需要操作文件或数据库，传统的写法需要在 finally 里手动关闭资源。从 Java 7 开始，提供了一个更优雅的语法糖：

```Java
// 把资源的创建写在 try 后面的括号里，Java 会自动帮你关闭它，不用再写 finally 了
try (Scanner sc = new Scanner(new File("data.txt"))) {
    // 读取文件的代码...
} catch (FileNotFoundException e) {
    e.printStackTrace();
}
```
