# 第一部分：前端数据传向后端

## 一、简单参数

形式：以**问号后接键值对**的形式将参数拼接到url地址中，通常是GET方式。

### @RequestParam接收

通过Spring提供的`@RequestParam`注解，将请求参数绑定给方法形参，示例：

```java
@DeleteMapping("/depts")
public Result delete(@RequestParam("id") Integer deptId){
    System.out.println("根据ID删除部门:"+ deptId);
    return Result.success();
}
```

**注意点：**

- 使用`@RequestParam`注解会自动进行类型转换，转换的类型就是方法形参的接收类型。

- 如果前后端参数名完全一致，`@RequestParam`可以省略；如果前后端参数名不一致，那就必须使用`@RequestParam`注解来指明后端接收的是哪个参数。

- 一旦在形参前面加了`@RequestParam`注解，那么就必须要传递这个参数，否则会报错（因为注解里有一个属性required默认为true）

- 如果既需要手动指定接收参数，又不强制要求这个参数必须传递，那么可以将`@RequestParam`的require属性置为false：

  ```java
  @RequestParam(value = “id”, required = false) //这样这个参数就可传可不传
  ```

- `@RequestParam`注解还可以给参数设置默认值，使用defaultValue属性，例如：

  ```java
  public Result query(@RequestParam(defaultValue = "1") Integer page){}
  ```

- 如果接收的是日期类型，需要指定日期格式，那么需要加上`@DateTimeFormat`注解，这个注解是Spring Boot项目的自带的注解不需要手动导入依赖。

  ```xml
  <dependency>
  	<groupId>org.springframework</groupId>
      <artifactId>spring-web</artifactId>
  </dependency>
  ```

  代码示例：

  ```java
  @GetMapping("/test")
  public Result test(@DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate begin){
      System.out.println(begin);
      System.out.println(begin.plusDays(1));
      return Result.success(begin);
  }
  ```

  

### 对象接收

如果简单路径传递的参数数量比较多，也可以定义一个专门的接收类Dto来接收参数，使用Dto对象来接收简单参数的规则与下面的用实体类对象接收JSON格式数据的规则一致，详情请参考：[对象接收](#requestBody)。



## 二、路径参数

### 单个参数

通过请求URL直接传递参数，使用{...}来标识该路径参数，需要使用`@PathVariable`注解来获取。

```java
@GetMapping("/depts/{id}")
public Result getInfo(@Pathvariable("id")Integer deptId){
	System.out.println("请根据ID查询部门数据:"+ deptId);
	Dept dept = deptService.getInfo(id);
	return Result.success(dept);
}
```



### 多个不同参数

在url中可以携带多个路径参数，如：/dept/1/0，接收方式如下：

```java
@GetMapping("/depts/{id}/{status}")
public Result getInfo(@PathVariable Integer id, @PathVariable Integer status){
	// 方法体...
}
```



### 多个同类参数

如果在url中传递多个同类型参数，如：/depts/1,2,3，接收方式如下：

```java
@DeleteMapping("/depts/{ids}")
public Result getInfo(@Pathvariable("ids")Integer[] deptIds){
	System.out.println("请根据ID查询部门数据:"+ deptIds);
	deptService.deleteBatch(ids);
	return Result.success();
}
```



## 三、JSON参数

### <a id = "requestBody" style="color:black">对象接收</a>

JSON格式的参数，通常会使用一个实体对象进行接收。

规则：JSON数据的键名与方法形参对象的属性名相同，并需要使用`@RequestBody`注解标识

```java
@GetMapping("/depts")
public Result getInfo(@RequestBody User user){
	deptService.updateById(user);
	return Result.success();
}
```

#### 注意事项

使用对象接收，要求前端传递的参数名与对象里的成员变量相同。如果不相同，那么有两种情况：

1. 如果前端有，成员变量里没有，那么这个参数无法被接收；

2. 如果成员变量里有，前端参数里没有，那么这个成员变量的值为默认值。

3. 如果前端参数和对象的属性名不一致，但又想要用此属性名来接收，那么可以此属性名上加注解`@JsonProperty`（前提是数据类型可以接收，只是名字不同）

   ```java
   import com.fasterxml.jackson.annotation.JsonProperty;
   
   public class ClueTrackRecord {
   	@JsonProperty("remark")
   	private String record;  //跟进记录
   }
   ```

4. 注意一旦打上了这个注解，就意味着前后端数据传递时，前端的remark视为与后端的record相等了，这个影响是双向的，既影响从前端到后端，也影响从后端到前端。（但是从后端到数据库的数据传递不受影响）

5. 如果请求体里有日期格式的属性，而接收对象又要指定它的日期格式的话，那么需要添加`@JsonFormat`注解，使用此注解前需要导入依赖jackson-annotation：

   ```xml
   <!--        JSON格式的时间转化为JAVA里的时间 -->
   <dependency>
   	<groupId>com.fasterxml.jackson.core</groupId>
   	<artifactId>jackson-annotations</artifactId>
   </dependency>
   ```
   
   代码示例：
   
   ```java
   // 经典Data类型，注意需要手动引入Jackson时间模块依赖
   
   import com.fasterxml.jackson.annotation.JsonFormat;
   import java.util.Date;
   
   // 格式化日期+时间 + 指定时区
   @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
   private Date createTime;
   
   // JDK8新时间类型
   @JsonFormat(pattern = "yyyy-MM-dd")
   private LocalDate begin;
   @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
   private LocalDateTime updateTime;
   ```

## 四、HttpServletRequest 对象获取

通过HttpServletRequest对象可以接收前端请求过来的参数，然后通过`对象.getParameter(变量名)`的方式获取请求参数，要注意以这种方式获得的参数全部是字符串类型。

   ```java
@DeleteMapping("/depts")
public Result delete(HttpServletRequest request){
    String idStr = request.getParameter("id");
    int id = Integer.parseInt(idstr);
    System.out.println("根据ID删除部门:"+id);
    return Result.success();
}
   ```

关于HttpServletRequest的API方法，可以参考文档：[07.ServletRequest 常用方法.md](07.ServletRequest 常用方法.md)



## 五、精炼小结

- 注解的参数中要指明从前端接收的是哪个参数，如果前后端参数名称相同，则可以不指定
- 对于对象接收，如果前端参数名与对象属性名不一致，那么可以使用`@JsonProperty("remark")`注解显性映射前后端参数名称
- 对于时间格式的接收，如果是在方法参数中接收，那么要加上`@DateTimeFormat(pattern = "yyyy-MM-dd")`注解，如果是在对象属性中接收，那么要加上`@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`注解
- 后端响应给前端的数据是对象，前端接收到的数据是JSON，它们之间的自动转换是通过`@RequestBody`和`@ResponseBody`自动实现的。
- 传给前端的不仅可以是一个对象，还可以是一个Map集合，同样也能自动转换为JSON格式传递给前端



# 第二部分：后端数据传向数据库

本笔记主要记录用Mybais实现Mapper接口时，Mapper层方法参数与Mapper注解中的占位符的对应问题

## 一、非对象类型形参

当方法形参是非对象类型时，Mapper注解直接用形参占位，示例：

```java
@Delete("delete from user where id = #{id}")
Integer deleteById(Integer id);
```



## 二、对象类型形参

当方法形参是对象类型时，Mapper注解使用对象的属性名占位，示例：

```java
@Insert("insert into user(username,password,name,age) values(#{username}),#{password},#{name},#{age}")
void insert (User user)
```



## 三、两个相同类型参数

当方法形参中有两个相同类型的参数时，需要使用`@Param`注解给接口方法的形参起名字，示例：

```java
@Select("select * from user where username=#{uname} and password=#{pwd}")
User findByUsernameAndPassword(@Param("uname")String username,@Param("pwd")String password);
```

说明：基于官方骨架创建的springboot项目中，接口编译时会保留方法形参名，`@Param`注解可以省略。

因为基于官网骨架创建的springboot项目的父工程配置了一个Maven编译的插件

```xml
<build>
	<pluginManagement>
    	<plugins>
        	<plugin>
            	<groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
 <!-- 当这个值为true时，在编译的时候会保留接口方法中的形参名称 -->
                	<parameters>true</parameters>
                </configuration>
            </plugin>
        </plugins>
    </pluginManagement>
</build>
```

如果是基于阿里云创建项目，就必须加上`@Param`注解



# 第三部分：数据库数据封装回后端

如果实体类属性名 和 数据库表查询返回的字段名一致，mybatis会自动封装。

如果实体类属性名和数据库查询返回的字段名不一致，那么就不能自动封装。

## 一、手动结果映射

通过`@Results`及`@Result`进行手动结果映射

```java
@Results({
	@Result(column = "create_time", property = "createTime"),
    @Result(column = "update_time", property = "updateTime")
})
@select("select id, name, create_time, update_time from dept order by update_time desc")
public List<Dept> findAll();
```



## 二、起别名

在SQL语句中，对不一样的列名起别名，别名和实体类属性名一样。

```java
@Select("select id, name, create_time createTime, update_time updateTime from dept ... ")
public List<Dept> findAll();
```



## 三、开启驼峰命名

如果字段名与属性名符号驼峰命名规则，mybatis会自动通过驼峰命名规则映射。

```yaml
mybatis:
	configuration:
		map-underscore-to-camel-case:true
```



## 四、主子表的封装

当涉及多表查询，且子表数据需要封装进主表时，普通的resultType自动封装就无法实现这个需求，这个时候就需要使用resultMap手动封装。

```xml
<!--  定义一个接收映射，设置它的id，以及封装的主表类  -->
<resultMap id="trackRecordMap" type="Clue">
    
    <!--  在主表类中，所有标签都有property和column这两个属性，前者是对象属性名，后者是数据库字段名  -->
	<id property="id" column="id"/>
	<result property="name" column="name"/>
	<result property="age" column="age"/>
    <!--  子表使用collection标签，property是主表的列表属性名，ofType是子表对应的类名  -->
	<collection property="trackRecords" ofType="ClueTrackRecord">
		<id property="id" column="tr_id"/>
		<result property="clueId" column="tr_clue_id"/>
		<result property="userId" column="tr_user_id"/>
		<result property="subject" column="tr_subject"/>
		<result property="assignName" column="assignName"/>
	</collection>
</resultMap>

<!--  正常写sql语句，但是注意不是用resultTpye，而是用resultMap接收，值为上述定义的resultMap的id  -->
<select id="findClueById" resultMap="trackRecordMap">
select c.id, c.name, c.age,
       ctr.id           tr_id,
       ctr.clue_id      tr_clue_id,
       ctr.user_id      tr_user_id,
       ctr.subject      tr_subject,
       u.name           assignName
from clue c
         left join clue_track_record ctr on c.id = ctr.clue_id
         left join user u on ctr.user_id = u.id
where c.id = #{id}
</select>
```

- 定义一个接收映射，设置它的id，以及封装的主表类
  - 在主表类中，除collection标签外，所有标签都有property和column这两个属性，前者是对象属性名，后者是数据库字段名
  - 子表使用collection标签，property是主表的列表属性名，ofType是子表对应的类名
- 正常写sql语句，但是注意不是用resultTpye，而是用resultMap接收，值为上述定义的resultMap的id
- 注意：多表联查时如果有相同字段名，一定要取别名来区分，虽然MySQL里面查出来的字段会包含表前缀，但是SQL查询结果的传到 java 时列名不包含表前缀。
