# Java 开发与工程实践入口

> 适用：Java、Spring、MyBatis、接口设计、数据库、缓存、Feign、性能、代码评审。

## 先读什么

- 接口与业务实现：项目既有 API 文档、DTO/VO、Service、Mapper、DDL。
- 数据库相关：先核对 DDL、迁移脚本、实体与 Mapper，不只凭实体类猜约束。
- 性能与分页：优先阅读 [[聚合查询、时间区间与批量数据补全]]。
- 高频状态与缓存：阅读 [[高频最新状态的Redis合并写模式]]。
- 公共微服务调用与降级：阅读 [[微服务公共API契约与Feign降级边界]]。

## 稳定工程原则

- 正确性、安全性优先；再考虑可维护性、抽象与性能优化。
- 聚合查询使用专用 DTO/Projection，不污染持久化实体。
- 时间范围默认左闭右开。
- 循环中避免随数据规模增长的数据库、缓存、MQ、Feign/HTTP 逐条调用；优先批量查询与 Map 回填，警惕 N+1。
- 需要强一致的关键状态以 MySQL 条件更新和事务为最终事实；缓存与消息用于加速和解耦。
- Swagger/Knife4j 项目中，DTO、VO 按项目风格补齐模型注解；路径参数显式使用 `@PathVariable("id")`。
- 默认构造器注入；项目使用 Lombok 且无额外构造逻辑时优先 `@RequiredArgsConstructor`。
