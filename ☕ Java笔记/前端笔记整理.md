# CSS部分

### H5复合选择器

#### 后代选择器

选中某元素的所有后代元素。

写法：用空格隔开

```css
div span{}
```

#### 子代选择器

选中某元素的直接后代元素。

写法：用 > 隔开

```css
div > span{}
```

#### 并集选择器

选中多组标签设置相同的样式。

作用：用于节省代码书写量，批量选中

```css
div,
p,
span{}
```



#### 交集选择器

选中同时满足多个条件的元素。

写法：选择器1选择器2{CSS属性}，选择器之间连写，没有任何符号。

```css
p.box{}
```



#### 伪类选择器

伪类表示元素状态，选中元素的某个状态设置样式。

鼠标悬停状态：选择器:hover{CSS属性}

| 选择器   | 作用           |
| -------- | -------------- |
| :link    | 访问前         |
| :visited | 访问后         |
| :hover   | 鼠标悬停       |
| :active  | 点击时（激活） |

写法示例：

```css
a:hover{}
```

```css
.box:hover{}
```



#### 属性选择器

元素名称[属性]{}

选择页面上type属性为text的<input>标签

```css
input[type=”text”]
```



### CSS三大特性：继承性、层叠性、优先级

**继承性**：子级默认继承父级的**文字控制属性**

**层叠性**：相同的属性会覆盖，后面的CSS属性覆盖前面的CSS属性；不同的属性会叠加。

**优先级**：也叫权重，当一个标签使用了多种选择器时，基于不同种类的选择器的匹配规则。

**优先级公式**：通配符选择器 < 标签选择器 < 类选择器 < id选择器 < 行内样式 < !important

**叠加计算**：如果是复合选择器，则需要权重叠加计算，从左向右依次比较个数，同一级个数多的优先级高，如果个数相同，则向后比较，不存在进位。

!important权重最高，继承权重最低。

## 函数声明

JS是弱类型语言，定义函数时不需要指定返回值类型和参数类型，且实参个数和形参个数可以不一致。eg:

function add(a,b){return a+b;}

#### 匿名函数：本质是定义一个没有名字的函数然后用一个变量来接收

```javascript
let fn = function(){}
```



#### 普通函数声明和匿名函数声明的区别：

| 对比                       | 普通函数声明                                                 | 箭头函数赋值                                 |
| -------------------------- | ------------------------------------------------------------ | -------------------------------------------- |
| 格式                       | function 函数名(参数列表){方法体}                            | const 方法名=(参数列表)=>{方法体}            |
| 变量提升                   | 整个函数会提升到代码最前面，编写者在定义前就能调用，不会报错 | 只有变量名提升，函数不提升，定义前调用会报错 |
| this指向                   | this会随调用环境变化                                         | this固定不变，继承外层作用域的this           |
| 能否用argument获取所有参数 | 可以用 arguments 获取所有参数                                | 不能用arguments                              |
| 能否当构造函数             | 可以 new 函数()                                              | 不能 new                                     |

**其它区别：**

- 箭头函数赋值的方式，可以用const/let/var三种方式接收。
- 当需要动态改变函数时，可以使用let。

- 当需要在定义前调用函数时，用function。

这也就是为什么Vue框架里面的函数声明习惯采用匿名函数的方式。



#### 立即执行函数：本质是定义一个匿名函数并立即执行

```javascript
(function(){})()
```

## JS核心语法-DOM

- ECMAScript：规定了JS基础语法核心知识，包括变量、数据类型、流程控制、函数、对象等。
- BOM：浏览器对象模型，用于操作浏览器本身，如页面弹窗、地址栏操作、关闭窗口等。
- DOM：文档对象模型，用于操作HTML文档，如：改变标签内的内容、改变标签内字体样式等。



DOM操作核心思想：将网页中所有的元素当作对象来处理。

| 对象      | 说明                                                         |
| --------- | ------------------------------------------------------------ |
| DOM       | Document Object Model文档对象模型，将标记语言的各个组成部分封装为对应的对象 |
| Document  | 整个文档对象                                                 |
| Element   | 元素对象                                                     |
| Attribute | 属性对象                                                     |
| Text      | 文本对象                                                     |
| Comment   | 注释对象                                                     |

获取DOM对象

- 根据CSS选择器来获取DOM元素，获取匹配到的第一个元素：document.querySelector(‘选择器’)
- 根据CSS选择器来获取DOM元素，获取匹配到的所有元素：document.querySelectorAll(‘选择器’)
- 注意：得到的是一个NodeList节点集合，是一个为数组（有长度、有索引的数组，但是没有数组操作的相关方法）

```javascript
let h1 = document.querySelector(‘#title1’)
h1.innerHTML = ‘修改后的文本内容’;
```



## JS监听事件

- 事件：HTML事件是发生在HTML元素上的“事情”。比如：按钮被点击、鼠标移动到元素上、按下键盘按键。

- 事件监听：JavaScript可以在事件触发时，就立即调用一个函数做出响应，也称为事件绑定或注册事件。

- 语法：事件源.addEventListener(‘事件类型’,事件触发执行的函数);

- 事件监听三要素：

  - 事件源：哪个DOM元素触发了事件，要获取DOM元素
  - 事件类型：用什么方式触发，比如鼠标单击click
  - 事件触发执行的函数：要做什么事

- 代码示例：

  ```html
  <input type="button" id="btn1" value="点我一下试试1">
  <input type="button" id="btn2" value="点我一下试试2">
  <script>
  	document.querySeletor(‘#btn1’).addEventListener(‘click’,function(){console.log(‘我被点击了’)})
  </script>
  ```

# Vue

Vue是一款用于构建用户界面的渐进式的JavaScript框架（官方：https://cn.vue.js.org）

**框架**：就是一套完整的项目解决方案，用于快速构建项目

**优点**：大大提升前端项目的开发效率

**缺点**：需要理解记忆框架的使用规则



### 常用指令

HTML标签上带有v-前缀的特殊属性，不同的指令具有不同的含义，可以实现不同的功能。

| 指令                  | 作用                                                |
| --------------------- | --------------------------------------------------- |
| v-for                 | 列表渲染，遍历容器的元素或者对象的属性              |
| v-bind                | 为HTML标签绑定属性值，如设置href，css样式等         |
| v-if/v-else-if/v-else | 条件性的渲染某元素，判定为true时渲染，否则不渲染    |
| v-show                | 根据条件展示某元素，区别在于切换的是display属性的值 |
| v-model               | 在表单元素上创建双向数据绑定                        |
| v-on                  | 为HTML标签绑定事件                                  |

注意事项：

- v-bind通常用冒号简写，v-on通常用`@`简写。

- 在HTML页面里面插入数据模型，需要使用插值表达式

- 在属性里面插入数据模型，不需要插值表达式

- 属性名前如果没有冒号，那么它的值表示一个普通的字符串或数字；如果属性前有冒号，那么这个属性是v-bind的简写，它的值是一个数据模型

  

### 插槽表达式

当表格显示数据不是后端传过来的数据，而是要经过处理后才能显示时，就要用到插槽，插槽的语法为：

```vue
<template default = #scope></template>
```

使用插槽的示例：

```vue
<el-table-column label="状态" align="center">
	<template #default="scope">
    	<el-tag :type="scope.row.status == 1 ? 'success':'danger'">
        	{{scope.row.status == 1 ? '启用':'停用'}}
        </el-tag>
    </template>
</el-table-column>
```

- scope.row.name => 拿到这一行的数据中变量名为name的值

- 在显示表格序号时，可以使用`type = "index"`，或者使用`scope.index + 1`，`type = "index"` 这个是唯一一个需要刷新才能在页面上显示的属性

# Ajax

- 介绍：Asynchronous JavaScript And XML，异步的JavaScript和XML。
- XML（Extensible Markup Language）可扩展标记语言，本质是一种数据格式，可以用来存储复杂的数据结构。
- 作用：
  - 数据交换：通过Ajax可以给服务器发送请求，并获取服务器响应的数据。
  - 异步交互：可以在不重新加载整个界面的情况下，与服务器交换数据并更新部分网页的技术，如：搜索联想、用户名是否可用的校验等等。



### 钩子函数

| 钩子              | 时机   | 作用                                                         | 能做什么                                                     |
| ----------------- | ------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **beforeCreate**  | 创建前 | 这是最早触发的钩子。此时组件实例刚刚被初始化，但 `data` 和 `methods` 等数据和方法都还没有准备好。 | 几乎很少用到。因为拿不到数据，也操作不了页面                 |
| **created**       | 创建后 | 此时组件的数据（`data`、`methods`）已经初始化完毕，但页面（DOM）还没渲染出来 | **非常常用！** 适合在这里发起异步请求（比如调用后端接口获取初始数据）、结束页面的 loading 加载状态 |
| **beforeMount**   | 挂载前 | 模板已经编译完成，但还没有真正地把内容替换到网页上           | 依然无法操作真实的 DOM 元素，实际开发中用得比较少            |
| **mounted**       | 挂载后 | 组件已经完全渲染到了页面上，你可以拿到真实的 DOM 节点了      | **极其常用！** 适合在这里操作 DOM、初始化一些依赖 DOM 的第三方插件（比如 ECharts 图表）、开启定时器或绑定事件监听 |
| **beforeUpdate**  | 更新前 | 你的数据已经变了，但页面还没来得及重新渲染                   | 可以在此刻获取更新前的旧 DOM 状态                            |
| **updated**       | 更新后 | 数据变了，页面也已经重新渲染完毕                             | 如果你需要根据最新的数据去操作 DOM，可以在这里进行。**注意：** 尽量不要在这里修改数据，否则很容易引发死循环 |
| **beforeDestroy** | 销毁前 | 组件即将被销毁，但此时此刻组件还是完好可用的                 | **非常重要！** 适合用来做“大扫除”——清除定时器（`clearInterval`）、解绑全局事件监听、取消未完成的网络请求，防止内存泄漏 |
| **destroyed**     | 销毁后 | 组件已经被彻底拆除，所有的事件监听器和子组件都被移除了       | 此时组件已经不存在了，一般不需要在这里写逻辑                 |
