## 双亲委派

防止串改和防止类的重复加载

当一个类要加载时;入口类加载器为AppClassLoader

但是AppClassloader不会立即加载,而是委托它的父类加载器

它的父类加载器 PlaFromClassLoader拿到之后,也不会立即加载,继续往上委托父类

最后由BootsstrapClassLoader检查是否由自己加载,

如果是,则加载

如果不是,则下传入给子类加载器

如果最后都无法加载,最后则抛异常 ClassNotfundExeption

# 反射

## 介绍:

框架技术的灵魂

是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法对于任意一个对象，都能够调用它的任意属性和方法 

这种动态获取信息以及动态调用对象方法的功能称为Java语言的反射机制

### 暴力反射

不推荐暴力反射 会破会封装原则

```java
//        创建Propperties集合,专门存入配置文件内容,配置文件一般是键值对的方式,所以这个集合是MAP集合,并且可以通过键来取值
        Properties prop = new Properties();
//        先创造这个类的字节码类对象,然后获取这个字节码类对象的类加载器,然后通过类加载器的方法取到配置文件的内容保存为流
        InputStream l = Demo09.class.getClassLoader().getResourceAsStream("fk.properties");
//        将取到的流存入到专门存配置文件内容的mpa集合中
        prop.load(l);
        System.out.println(prop);

//   通过KEY值找到这个配置文件中的类名,并且用这个类名创建一个字节码类
        Class<?> className = Class.forName(prop.getProperty("className"));
//        通过字节码类来创建一个构造器对象并且用这个构造器对象创造一个对象;
        Object o = className.getDeclaredConstructor().newInstance();

//  通过key来找到配置文件集合中的值,也就是方法名,然后用这个方法名创建一个方法名对象
        Method method = className.getMethod(prop.getProperty("methodName"));
//        使用方法名对象的方法,传入对象,就能使用该方法.
        System.out.println(method.invoke(o));
```



# 注解

注解的作用:对程序就行标注

注释式给人开的,注解是给虚拟机看的

设提供注解,谁解析

注解定义出来一定要有人解析,不然写了没有一点用

#### 需要了解的点:

注解用在什么上面

注解有什么效果

#### 注意:

注解中的成员,称之为属性

必须要有返回值

有多个属性,且没有默认值,就必须给每个属性赋值

如果属性类型是数组,且只赋一个数据,则大括号可以不写,如果想赋值多个,则大括号必须写

如果属性名为value,且只给它赋值,value可以不写
