## 02-Python基础
## 1 面向对象
### 1.1 类与对象
Python是面向对象的编程语言，采用类（class）来生产对象
**类**：对象的模板，定义了对象的属性和方法
**对象**：类的实例，是具体的个体
#### 1.1.1 基础语法
定义类和创建对象的基础语法如下：
```
  Python
  # 1）创建类
  **class** **类名**:
  """类的说明；文档字符串"""
  # 构造方法（初始化）
  **def** **\_\_init\_\_**(self, 属性形参列表):
  self.实例属性 = 形参 # 实例属性
  # 实例方法
  **def** **方法名**(self):
  print(f"{self.实例属性} 。。。")
  # 2）创建对象,调用变量和方法
  对象 = 类名(参数列表)
  对象.变量
  对象.方法名()
```
具体的代码示例：
```
  Python
  01-基础语法
  """
  #1) 创建类
  class 类名:
  # 构造方法（初始化）
  def \_\_init\_\_(self, 属性形参列表):
  self.实例属性 = 形参 # 实例属性
  # 实例方法
  def 方法名(self):
  print(f"{self.实例属性} 。。。")
  #2) 创建对象,调用变量和方法
  对象 = 类名(参数列表)
  对象.变量
  对象.方法名()
  """
  # 1) 创建一个猫类, 定义name和age属性, 定义say方法
  class Cat:
  """猫类"""
  # 构造方法, 定义类的属性
  def \_\_init\_\_(self, name, age):
  self.name = name
  self.age = age
  # self 代表的就是当前对象, 相当于Java中的this关键字
  def say(self):
  print(f"我是一只猫,我的名字叫:{self.name},我的年龄是:{self.age}")
  # 2) 创建对象，调用对象的属性和方法
  cat = Cat("花小喵", 1)
  # cat.name = "花小喵"
  # cat.age = 2
  cat.say()
```
#### 1.1.2 属性
在Python中属性有两种：
类属性: 定义在类中方法外，会被类的所有实例对象所共享（可通过 类名.属性名 或 实例名.属性名 访问；可通过 类名.属性名 修改）
实例属性: 定义在类的方法中，为每个实例对象独享（可以通过 实例名.属性名 访问或修改）
```
  Python
  02-属性
  """
  在Python中属性有两种：
  类属性: 定义在类中方法外，会被类的所有实例对象所共享（可通过 类名.属性名或 实例名.属性名 访问；可通过 类名.属性名 修改）
  实例属性: 定义在类的方法中，为每个实例对象独享（可以通过 实例名.属性名 访问或修改）
  """
  class Person:
  # 类属性: 定义在类中方法外
  age = 10
  def \_\_init\_\_(self):
  # 实例属性: 定义在类的方法中
  self.name = None
  # 类属性: 定义在类中方法外，会被类的所有实例对象所共享
  p1 = Person()
  p2 = Person()
  # 可通过 类名.属性名或 实例名.属性名 访问
  print(Person.age)
  print(p1.age)
  print(p2.age)
  # 可通过 类名.属性名 修改
  Person.age = 20
  # p1.age = 30
  print(Person.age)
  print(p1.age)
  print(p2.age)
  print("\*" \* 40)
  # 实例属性: 定义在类的方法中，为每个实例对象独享（可以通过 实例名.属性名 访问或修改）
  print(f"p1.name:{p1.name}")
  print(f"p2.name:{p2.name}")
  p1.name = "张三"
  print(f"p1.name:{p1.name}")
  print(f"p2.name:{p2.name}")
```
#### 1.1.3 方法
在Python中方法主要有两种：
类方法：在类中通过 \@classmethod 定义，第一个参数为cls，代表类本身，可以通过 类名.方法名 或者 实例名.方法名（不推荐）调用
实例方法：在类中定义，第一个参数为self，代表实例本身，只能通过 实例名.方法名 调用
注意：实例方法中可以访问类方法、属性以及实例方法、属性；但是类方法只能访问类方法、属性 (类只能访问类; 实例可以访问一切)
```
  Python
  03-方法
  """
  在Python中方法主要有两种：
  类方法：在类中通过 \@classmethod 定义，第一个参数为cls，代表类本身，可以通过类名.方法名 或者 实例名.方法名 （不推荐）调用
  实例方法：在类中定义，第一个参数为self，代表实例本身，只能通过 实例名.方法名 调用
  注意：实例方法中可以访问类方法、属性以及实例方法、属性；但是类方法只能访问类方法、属性
  """
  class Person:
  # 类属性
  age = 10
  def \_\_init\_\_(self):
  self.name = "张三"
  # 类方法：在类中通过 \@classmethod 定义，第一个参数为cls，代表类本身
  \@classmethod
  def m1(cls):
  print(f"m1方法{cls.age}")
  # 实例方法：在类中定义，第一个参数为self，代表实例本身
  def m2(self):
  print(f"m2方法:{self.name}")
  # 类方法：可以通过类名.方法名 或者 实例名.方法名 （不推荐）调用
  # Person.m1() # 类名.方法名
  # Person().m1() # 实例名.方法名 （不推荐）
  # 实例方法：只能通过 实例名.方法名 调用
  Person().m2()
```
#### 1.1.4 魔术方法
Python提供了许多以 \_\_ 开头和结尾的**魔术方法** （也称为 双下方法），他们会在对象生命周期的某个位置自动调用。常用的有：
\_\_init\_\_()：对象在初始化时自动调用的方法
\_\_str\_\_()：在打印对象时自动调用的方法 类似于Java的toString()
\_\_eq\_\_()：在比较两个对象是否相等时自动调用的方法 类似于Java的equals()
```
  Python
  04-魔术方法
  """
  Python提供了许多以 \_\_ 开头和结尾的魔术方法 （也称为 双下方法），他们会在对象生命周期的某个位置自动调用。常用的有：
  \_\_init\_\_()：对象在初始化时自动调用的方法
  \_\_str\_\_()：在打印对象时自动调用的方法
  \_\_eq\_\_()：在比较两个对象是否相等时自动调用的方法
  """
  class Book:
  # \_\_init\_\_()：对象在初始化时自动调用的方法
  def \_\_init\_\_(self, name, price):
  self.name = name
  self.price = price
  # \_\_str\_\_()：在打印对象时自动调用的方法
  def \_\_str\_\_(self):
  return f"Book:name-{self.name}, price-{self.price}"
  # \_\_eq\_\_()：在比较两个对象是否相等时自动调用的方法
  def \_\_eq\_\_(self, other):
  return self.name == other.name
  b1 = Book("西游记", 10)
  b2 = Book("西游记", 100)
  print(b1)
  print(b2)
  print(f"b1==b2:{b1 == b2}")
```
### 1.2 封装
#### 1.2.1 私有化
如果我们想类中的某些属性和方法只能在类内访问，外部无法访问，可以将其私有化，具体实现方式是：
\_\_属性名：使用双下划线修饰一个属性的名字
\_\_方法名：使用双下划线修饰一个方法的名字
注意：
使用一个下划线修饰的属性或者方法名被视为非公开访问内容，不建议外部访问，但是只是建议，不是强制；双下划线是强制
双下划线实现强制外部不能访问的原理是名称改写，例如\_\_属性，会被改写为_类名\_\_属性
此时只能在类内部可以通过\_\_属性访问，其他地方无法访问或通过 \_类名\_\_属性访问
如果想要访问私有化的属性和方法, 可以在类中提供公开的方法间接访问
```
  Python
  05-封装-私有化
  """
  如果我们想类中的某些属性和方法只能在类内访问，外部无法访问，可以将其私有化，具体实现方式是：
  \_\_属性名：使用双下划线修饰一个属性的名字
  \_\_方法名：使用双下划线修饰一个方法的名字
  注意：
  1. 使用一个下划线修饰的属性或者方法名被视为非公开访问内容，不建议外部访问，但是只是建议，不是强制；双下划线是强制
  2. 双下划线实现强制外部不能访问的原理是名称改写，例如\_\_属性，会被改写为_类名\_\_属性
  此时只能在类内部可以通过\_\_属性访问，其他地方无法访问或通过 \_类名\_\_属性访问
  3. 如果想要访问私有化的属性和方法, 可以在类中提供公开的方法间接访问
  """
  class User:
  def \_\_init\_\_(self):
  # \_\_属性名：使用双下划线修饰一个属性的名字
  self.\_\_name = "张三"
  # get方法获取name属性的值
  def get_name(self):
  return self.\_\_name
  # set方法设置name的值
  def set_name(self, name):
  self.\_\_name = name
  user = User()
  user.set_name("李四")
  print(user.get_name())
```
#### 1.2.2 装饰器
装饰器其实本质上就是一个注解，标注在属性和方法上，对原有功能进行改造或者增强
\@property：将一个方法转换为属性来调用，转换后可直接使用 .方法名 来使用，而无需使用 .方法名()
\@方法名.setter：将方法名设置为去掉双下划线的私有属性名
```
  Python
  06-封装-装饰器
  """
  装饰器其实本质上就是一个注解，标注在属性和方法上，对原有功能进行改造或者增强
  \@property：将一个方法转换为属性来调用，转换后可直接使用 .方法名 来使用，而无需使用 .方法名()
  \@方法名.setter：将方法名设置为去掉双下划线的私有属性名
  我们使用这两个注解来完成一个类中属性的set/get实现(注意:get方法必须在set之前)
  """
  class User:
  def \_\_init\_\_(self, name, age):
  self.\_\_name = name
  self.\_\_age1 = age
  \@property
  def age(self):
  return self.\_\_age
  # set
  \@age.setter
  def age(self, age):
  self.\_\_age = age
  # \@property：将一个方法转换为属性来调用，转换后可直接使用 .方法名 来使用，而无需使用 .方法名()
  \@property
  def name(self):
  return self.\_\_name
  # \@方法名.setter：将方法名设置为去掉双下划线的私有属性名
  \@name.setter
  def name(self, name):
  if len(name) \> 5:
  print("名字长度不能超过5个字符")
  return
  self.\_\_name = name
  user = User("张三", 17)
  user.name = "李四"
  print(user.name)
  user.age = 18
  print(user.age)
```
### 1.3 继承
在python中，子类可以继承父类的属性和方法，实现代码复用
#### 1.3.1 单继承
单继承指的是一个类只继承一个父类，语法：class 子类名（父类名），它的特点如下：
子类可以使用父类的非私有成员
子类可以重写父类的方法，调用方法时先在子类中查找，如果子类中没有，则会去父类中查找
子类可以使用 super().方法名() 或 父类名.方法名() 来调用父类的方法
子类重写\_\_init\_\_()并调用时，不会执行父类的\_\_init\_\_()方法。如有必要，需手动调用
```
  Python
  07-继承-单继承
  """*
  *单继承指的是一个类只继承一个父类，语法：class 子类名（父类名），它的特点如下：*
  *1. 子类可以使用父类的非私有方法*
  *2. 子类可以重写父类的方法，调用方法时先在子类中查找，如果子类中没有，则会去父类中查找*
  *3. 子类可以使用 super().方法名() 或 父类名.方法名() 来调用父类的方法*
  *4. 子类重写\_\_init\_\_()并调用时，不会执行父类的\_\_init\_\_()方法。如有必要，需手动调用*
  *"""*
  *class Animal:*
  * def \_\_init\_\_(self, name):*
  * self.name = name*
  * def eat(self):*
  * print(f'{self.name}在吃饭饭')*
  * def sleep(self):*
  * print(f'{self.name}在睡觉觉')*
  *class Dog(Animal):*
  * def \_\_init\_\_(self):*
  * super().\_\_init\_\_("小黑")*
  * def eat(self):*
  * print(f'{self.name}在吃骨头')*
  * super().sleep()*
  * # Animal.sleep(self)*
  *d = Dog()*
  *d.eat()
```
#### 1.3.2 多继承
多继承指的是一个类可以继承多个父类，语法：class 子类名（父类1，父类2，\...\....），它的特点如下：
子类可以使用所有父类的非私有成员
调用方法时先在子类中查找，如果子类中没有，若不存在则从左到右依次查找父类中是否包含方法
可使用 类名.\_\_mro\_\_ 访问类的继承链来查看方法解析顺序,object是所有类的直接或间接父类
```
  Python
  08-继承-多继承
  *"""*
  *多继承指的是一个类可以继承多个父类，语法：class 子类名（父类1，父类2，\...\....），它的特点如下：*
  *1. 子类可以使用所有父类的非私有成员*
  *2. 调用方法时先在子类中查找，如果子类中没有，若不存在则从左到右依次查找父类中是否包含方法*
  *3. 可使用 类名.\_\_mro\_\_ 访问类的继承链来查看方法解析顺序,object是所有类的直接或间接父类*
  *"""*
  class A(object):
  def m(self):
  print("ma")
  class B(object):
  def m(self):
  print("mb")
  class C(A,B):
  pass
  c = C()
  c.m()
  print(C.\_\_mro\_\_)
```
### 1.4 多态
Python的多态是一种伪多态，它不关心对象的具体类型，只关心对象是否具有我们需要的方法或属性
也就是说：只要对象支持预期的行为（即有相应的接口），就可以认为它是合适的
```
  Python
  09-多态
  """
  Python的多态是一种伪多态，它不关心对象的具体类型，只关心对象是否具有我们需要的方法或属性
  也就是说：只要对象支持预期的行为（即有相应的接口），就可以认为它是合适的
  """
  # 动物
  class Animal:
  def speak(self):
  pass
  # 狗
  class Dog(Animal):
  def speak(self):
  print("汪汪汪")
  class Desk:
  def speak(self):
  print("吱吱吱")
  class Water:
  pass
  # Python的多态是一种伪多态，它不关心对象的具体类型，只关心对象是否具有我们需要的方法或属性
  def test(animal: Animal):
  animal.speak()
  # test(Dog())
  # test(Desk())
  test(Water())
```
